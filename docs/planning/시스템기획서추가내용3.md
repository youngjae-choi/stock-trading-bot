있습니다. 다만 **“무료 외부 LLM으로 금융분석까지 맡긴다”**는 표현은 조금 나눠서 봐야 합니다.  
무료 LLM도 뉴스 요약, 공시 요약, 시장 분위기 분류, 종목 선정 사유 정리 같은 **분석 보조**는 충분히 가능합니다. 하지만 **실시간 매수/매도 최종 판단**이나 **리스크 룰 변경**은 무료든 유료든 LLM에게 직접 맡기면 안 됩니다. 그건 Python 룰 엔진과 Risk Guard가 해야 합니다.

---

## 1. 무료로 쓸 만한 외부 LLM 후보

아래 후보들은 “금융 특화 모델”이라기보다는, **금융 데이터를 넣어주면 분석 보조가 가능한 범용 LLM**에 가깝습니다.  
즉, 모델이 스스로 최신 시세를 아는 게 아니라, 우리가 KIS/DART/뉴스 API에서 데이터를 넣어줘야 제대로 작동합니다.

| **후보** | **무료 사용 가능성** | **금융분석 적합도** | **추천 용도** |
|---|---:|---:|---|
| **Google Gemini API** | 있음 | 높음 | 뉴스/공시/해외장 요약, 장문 분석 |
| **Groq API** | 있음 | 중간~높음 | 빠른 뉴스 분류, 감성 분류, 후보 사유 요약 |
| **OpenRouter Free Models** | 있음 | 중간 | 무료 모델 백업, 보조 의견, 교차검증 |
| **Hugging Face Inference Providers** | 제한적 무료/모델별 | 중간 | 오픈소스 모델 테스트, 특화 모델 실험 |
| **Cloudflare Workers AI** | 무료/저비용 가능 | 중간 | 경량 분류, 요약, 서버리스 연결 |
| **로컬 오픈소스 LLM** | API 비용 없음 | 모델에 따라 다름 | 비용 절감, 내부 로그 요약 |

무료 LLM API 목록과 공급자 비교 자료는 여러 공개 리소스에서 정리되어 있고, OpenRouter나 Hugging Face처럼 여러 모델을 한 API에서 선택할 수 있는 플랫폼도 있습니다. 다만 무료 한도와 사용 가능 모델은 자주 바뀌므로 실제 운영 전에는 각 플랫폼의 현재 quota를 확인해야 합니다. [1] [4] [5] [8]

## 2. 제일 현실적인 추천 조합

자동매매 시스템 기준으로는 이렇게 쓰는 게 좋습니다.

### 2.1 1순위: Gemini 무료 API

Gemini는 무료 한도가 있는 편이고, 특히 **긴 문서 요약**과 **뉴스 묶음 정리**에 잘 맞습니다.

추천 용도:

```text
- 전일 미국장 요약
- 국내 주요 뉴스 요약
- DART 공시 요약
- 섹터별 이슈 정리
- 장마감 복기 보조
```

비추천 용도:

```text
- 실시간 매수/매도 판단
- 주문 직전 최종 판단
- 손절/익절 조건 변경
```

Gemini는 사용자님 시스템에서는 **“야간 글로벌 브리핑 담당”**으로 가장 좋습니다.

---

### 2.2 2순위: Groq 무료 API

Groq는 장점이 명확합니다. **빠릅니다.**  
그래서 깊은 전략 판단보다는 짧고 빠른 분석에 좋습니다.

추천 용도:

```text
- 뉴스 제목 100개 빠른 분류
- 긍정/부정/중립 감성 분류
- 종목별 이슈 태깅
- 후보 종목 선정 사유 한 줄 요약
- 장중 로그 요약
```

예를 들어:

```text
뉴스 제목: 삼성전자, HBM 공급 확대 기대감
결과:
{
  "sector": "반도체",
  "sentiment": "positive",
  "impact": "medium",
  "reason": "HBM 공급 확대 기대는 반도체 대형주 수급에 긍정적"
}
```

Groq는 “빠른 분류기”로 쓰면 좋습니다.  
Opus/GPT 같은 메인 판단 모델의 비용을 줄여주는 역할입니다.

---

### 2.3 3순위: OpenRouter Free Models

OpenRouter는 여러 모델을 한 API 방식으로 쓸 수 있는 장점이 있습니다.  
무료 모델도 제공되는 경우가 있어서, **백업 LLM 라우터**로 쓰기 좋습니다.

추천 용도:

```text
- Gemini quota 초과 시 대체 요약
- Groq 장애 시 백업 분류
- 무료 모델 간 교차검증
- 같은 뉴스에 대한 second opinion
```

예를 들어:

```text
Gemini: 미국장 요약
OpenRouter 무료 모델: 동일 내용 보조 요약
시스템: 두 요약의 공통 섹터만 내일 관심 섹터 후보로 채택
```

OpenRouter는 모델 목록이 수시로 바뀌므로, 현재 무료로 제공되는 모델을 확인한 뒤 연결하는 방식이 좋습니다. [8]

---

### 2.4 4순위: Hugging Face Inference Providers

Hugging Face는 오픈소스 모델 실험에 좋습니다.  
특히 금융 특화 모델이나 한국어 모델을 테스트할 때 유용합니다.

추천 용도:

```text
- 금융 특화 오픈소스 모델 테스트
- 한국어 뉴스 분류 모델 테스트
- 자체 파인튜닝 모델 배포 전 실험
- 감성분석/문서분류 모델 연결
```

다만 Hugging Face는 모델마다 속도, 안정성, 무료 한도, 제공 방식이 다를 수 있습니다.  
운영용이라기보다는 **실험실**에 가깝게 보는 게 좋습니다. [5]

---

## 3. 금융분석 기준으로 보면 누가 제일 낫나

“무료 또는 무료 한도”만 놓고 보면 이렇게 정리할 수 있습니다.

| **순위** | **모델/플랫폼** | **추천 점수** | **이유** |
|---:|---|---:|---|
| 1 | **Gemini 무료 API** | 8.5 / 10 | 장문 뉴스/해외장 요약에 강함 |
| 2 | **Groq 무료 API** | 7.5 / 10 | 빠른 분류/요약에 좋음 |
| 3 | **OpenRouter 무료 모델** | 7.0 / 10 | 백업/교차검증용으로 유용 |
| 4 | **Hugging Face** | 6.5 / 10 | 특화 모델 실험에 좋지만 운영 안정성은 모델별 차이 |
| 5 | **Cloudflare Workers AI** | 6.0 / 10 | 경량 작업, 서버리스 연결에 적합 |

제 추천은 이겁니다.

```text
Gemini = 무료 장문 분석 담당
Groq = 무료 고속 분류 담당
OpenRouter = 무료 백업/교차검증 담당
Hugging Face = 금융 특화 모델 실험 담당
```

## 4. 자동매매 시스템에 넣는 구조

무료 LLM은 **메인 엔진**이 아니라 **비용 절감형 보조 분석 계층**으로 넣어야 합니다.

```text
KIS / DART / 뉴스 API
        ↓
정량 데이터 정리: Python
        ↓
무료 LLM 1차 분석
- Gemini: 긴 뉴스 요약
- Groq: 빠른 분류
- OpenRouter: 백업 의견
        ↓
유료 LLM 또는 메인 LLM
- Opus: 시장 톤 최종 해석
- GPT: RulePack 구조화
        ↓
시스템 검증
- JSON Schema
- Risk Guard
- Runtime Validator
        ↓
Python 룰 엔진 실행
```

핵심은 이겁니다.

> 무료 LLM이 “분석 재료”를 만들고,  
> 유료 LLM 또는 시스템이 “실행 가능한 룰”로 정리하고,  
> 최종 판단은 Risk Guard가 검증한다.

## 5. 무료 LLM별 추천 역할

### 5.1 Gemini

```text
역할:
- 미국장 요약
- 장문 뉴스 요약
- 공시 요약
- 복기 로그 요약

사용 시점:
- 08:00
- 16:00
- 22:00
```

Gemini는 호출 수를 아껴서 하루 5~10회 정도만 쓰는 게 좋습니다.

---

### 5.2 Groq

```text
역할:
- 뉴스 빠른 분류
- 종목별 이슈 태깅
- 긍정/부정/중립 분류
- 짧은 매매 로그 요약

사용 시점:
- 장전
- 장중
- 장마감
```

Groq는 속도가 좋아서 **수백 개 뉴스 제목을 빠르게 분류**하는 데 적합합니다.

---

### 5.3 OpenRouter 무료 모델

```text
역할:
- Gemini 실패 시 대체
- Groq 실패 시 대체
- 무료 모델 간 비교
- 저비용 실험
```

OpenRouter는 라우터로 쓰면 좋습니다.  
즉, 모델을 하나 고정하기보다 **“무료로 가능한 모델 중 현재 되는 것”**을 선택하는 백업 채널입니다.

---

### 5.4 Hugging Face

```text
역할:
- 금융 감성분석 모델 실험
- 한국어 뉴스 분류 모델 실험
- 자체 모델 후보 테스트
```

Hugging Face는 장기적으로 “우리만의 금융 특화 모델”을 만들 때 좋습니다.

## 6. 샘플 코드: 무료 LLM 라우터

아래는 무료/저비용 LLM을 **순서대로 시도하는 라우터 예시**입니다.  
실제 API 키와 모델명은 가입한 플랫폼의 현재 모델명으로 바꿔야 합니다.

```python
import os
import requests
from datetime import datetime
from zoneinfo import ZoneInfo


class FreeLLMRouter:
    def __init__(self):
        self.providers = [
            {
                "name": "gemini",
                "enabled": bool(os.getenv("GEMINI_API_KEY")),
                "daily_limit": 10
            },
            {
                "name": "groq",
                "enabled": bool(os.getenv("GROQ_API_KEY")),
                "daily_limit": 50
            },
            {
                "name": "openrouter",
                "enabled": bool(os.getenv("OPENROUTER_API_KEY")),
                "daily_limit": 30
            }
        ]

        self.usage = {
            "date": self.today_key(),
            "gemini": 0,
            "groq": 0,
            "openrouter": 0
        }

    def today_key(self):
        now = datetime.now(ZoneInfo("Asia/Seoul"))
        return now.strftime("%Y-%m-%d")

    def can_use(self, provider):
        name = provider["name"]

        if not provider["enabled"]:
            return False

        if self.usage.get(name, 0) >= provider["daily_limit"]:
            return False

        return True

    def analyze_finance_text(self, task, text):
        prompt = f"""
너는 자동매매 시스템의 금융 분석 보조 AI다.

주의:
- 투자 조언이 아니라 분석 보조 결과만 작성한다.
- 매수/매도 지시는 하지 않는다.
- 입력 데이터에 없는 사실을 만들지 않는다.
- 결과는 JSON으로 작성한다.

작업:
{task}

입력:
{text}

출력 JSON 형식:
{{
  "summary": "...",
  "market_impact": "positive|neutral|negative|mixed",
  "related_sectors": ["..."],
  "risk_factors": ["..."],
  "confidence": 0.0
}}
"""

        for provider in self.providers:
            if not self.can_use(provider):
                continue

            try:
                if provider["name"] == "groq":
                    result = self.call_groq(prompt)
                elif provider["name"] == "openrouter":
                    result = self.call_openrouter(prompt)
                elif provider["name"] == "gemini":
                    result = self.call_gemini_placeholder(prompt)
                else:
                    continue

                self.usage[provider["name"]] += 1

                return {
                    "ok": True,
                    "provider": provider["name"],
                    "result": result
                }

            except Exception as e:
                print(f"{provider['name']} failed:", str(e))
                continue

        return {
            "ok": False,
            "provider": "none",
            "result": {
                "summary": "사용 가능한 무료 LLM이 없습니다.",
                "market_impact": "neutral",
                "related_sectors": [],
                "risk_factors": ["LLM 분석 실패"],
                "confidence": 0.0
            }
        }

    def call_groq(self, prompt):
        api_key = os.getenv("GROQ_API_KEY")

        url = "https://api.groq.com/openai/v1/chat/completions"

        payload = {
            "model": "llama-3.1-8b-instant",
            "messages": [
                {
                    "role": "system",
                    "content": "너는 금융 뉴스 분류와 요약을 돕는 분석 보조 AI다."
                },
                {
                    "role": "user",
                    "content": prompt
                }
            ],
            "temperature": 0.2
        }

        headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }

        response = requests.post(url, json=payload, headers=headers, timeout=30)
        response.raise_for_status()

        data = response.json()
        return data["choices"][0]["message"]["content"]

    def call_openrouter(self, prompt):
        api_key = os.getenv("OPENROUTER_API_KEY")

        url = "https://openrouter.ai/api/v1/chat/completions"

        payload = {
            "model": "openrouter/auto",
            "messages": [
                {
                    "role": "system",
                    "content": "너는 금융 데이터 분석 보조 AI다."
                },
                {
                    "role": "user",
                    "content": prompt
                }
            ],
            "temperature": 0.2
        }

        headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json",
            "HTTP-Referer": "http://localhost",
            "X-Title": "Dantabot Console"
        }

        response = requests.post(url, json=payload, headers=headers, timeout=30)
        response.raise_for_status()

        data = response.json()
        return data["choices"][0]["message"]["content"]

    def call_gemini_placeholder(self, prompt):
        # 실제 Gemini SDK 또는 REST API 연결부로 교체
        # 여기서는 라우터 구조를 보여주기 위한 placeholder
        return """
{
  "summary": "Gemini 분석 결과가 여기에 들어갑니다.",
  "market_impact": "mixed",
  "related_sectors": ["반도체", "AI"],
  "risk_factors": ["환율 변동", "미국 기술주 변동성"],
  "confidence": 0.72
}
"""


if __name__ == "__main__":
    router = FreeLLMRouter()

    sample_news = """
미국 나스닥은 대형 기술주 중심으로 상승했다.
반도체주는 AI 서버 수요 기대감으로 강세를 보였고,
달러 강세와 국채금리 상승은 일부 부담 요인으로 작용했다.
"""

    result = router.analyze_finance_text(
        task="내일 한국장에 영향을 줄 수 있는 섹터와 리스크를 요약해줘.",
        text=sample_news
    )

    print(result)
```

## 7. 코드 설명

이 코드는 무료 LLM을 하나만 믿지 않고, **여러 무료 공급자를 순서대로 시도**합니다.

| **구성** | **설명** |
|---|---|
| `FreeLLMRouter` | 무료 LLM 호출을 관리하는 라우터 |
| `providers` | Gemini, Groq, OpenRouter 사용 가능 여부와 일일 한도 관리 |
| `analyze_finance_text()` | 금융 뉴스/공시/로그 분석 요청 |
| `call_groq()` | Groq API 호출 |
| `call_openrouter()` | OpenRouter API 호출 |
| `call_gemini_placeholder()` | Gemini 연결 위치 |
| `daily_limit` | 자체 호출량 제한 |

이 구조의 장점은 단순합니다.

```text
Gemini quota 초과 → Groq 사용
Groq 실패 → OpenRouter 사용
OpenRouter 실패 → 중립 결과 반환
```

즉, 무료 API가 막혀도 자동매매 시스템 전체가 멈추지 않습니다.

## 9. 프론트 콘솔에 추가하면 좋은 화면

지금 만든 `단타봇.html`에 나중에 이런 섹션을 넣으면 좋습니다.

```text
AI Provider Status

Gemini
- Role: Global Brief / Long Summary
- Daily Usage: 3 / 10
- Status: Available

Groq
- Role: Fast Classification
- Daily Usage: 18 / 50
- Status: Available

OpenRouter
- Role: Backup LLM
- Daily Usage: 2 / 30
- Status: Standby

Hugging Face
- Role: Model Lab
- Status: Test Only
```

그리고 `AI RulePack` 화면에는 이렇게 남기면 됩니다.

```text
오늘 RulePack 생성에 사용된 AI Source
- Global Brief: Gemini
- News Classification: Groq
- Backup Opinion: OpenRouter
- RulePack Structuring: GPT
- Final Validation: System
```

이렇게 해야 나중에 “왜 이런 판단이 나왔는지” 추적할 수 있습니다.

## 10. 최종 추천

무료 외부 LLM을 쓴다면 이 조합이 가장 현실적입니다.

```text
1순위: Gemini
- 장문 뉴스, 미국장, 공시 요약

2순위: Groq
- 빠른 뉴스 분류, 감성 분류, 종목 이슈 태깅

3순위: OpenRouter 무료 모델
- 백업, 교차검증, 실험

4순위: Hugging Face
- 금융 특화 오픈소스 모델 테스트
```

자동매매 시스템에서는 이렇게 정리하면 됩니다.

```text
무료 LLM = 분석 보조
유료 LLM = 고급 판단/구조화
Python 룰 엔진 = 실제 실행
Risk Guard = 최종 안전장치
```

즉, 무료 LLM만으로도 **뉴스 분석, 섹터 분류, 공시 요약, 복기 보조**는 충분히 가능합니다.  
하지만 **매매 실행 결정은 절대 무료 LLM에 직접 맡기지 않고**, 검증 가능한 RulePack과 Python 룰 엔진으로 처리하는 구조가 맞습니다.
