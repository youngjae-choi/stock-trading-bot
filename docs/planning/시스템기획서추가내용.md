맞습니다. **복기의 진짜 목적은 “리포트 보기”가 아니라 다음 매매를 더 잘하게 만드는 학습 루프를 만드는 것**입니다.  
자동매매 시스템이 계속 좋아지려면 “오늘 얼마 벌었나?”보다 더 중요한 것을 저장해야 합니다. 바로 **어떤 조건이 좋은 타이밍을 만들었고, 어떤 조건 조합이 나쁜 진입을 만들었는지**입니다.

다만 핵심 원칙이 있습니다.

> **자동학습은 가능하지만, 실시간으로 자기 마음대로 전략을 바꾸게 하면 안 됩니다.**  
> 학습은 장마감 후 수행하고, 다음날 RulePack에 반영하는 방식이 안전합니다.

즉, 시스템은 매일 거래를 하면서 경험을 쌓고, 장마감 후 좋은 전략·나쁜 전략·좋은 타이밍·나쁜 타이밍을 분류한 뒤, 다음날 AI RulePack 생성에 반영해야 합니다.

---

## 1. 자동학습의 핵심 개념

자동학습은 단순히 “수익 난 전략을 기억한다”가 아닙니다.  
수익이 났더라도 운이었을 수 있고, 손실이 났더라도 좋은 판단이었을 수 있습니다.

따라서 학습 대상은 크게 5가지입니다.

| **학습 대상** | **의미** |
|---|---|
| 좋은 진입 조건 | 어떤 조건 조합에서 진입 후 유리하게 움직였는가 |
| 나쁜 진입 조건 | 어떤 조건 조합에서 손절/시간손절이 자주 발생했는가 |
| 좋은 청산 조건 | 익절, 트레일링, 시간손절 중 무엇이 효과적이었는가 |
| 나쁜 시간대 | 특정 시간대에 성과가 낮았는가 |
| 시장 국면 | 강세장/약세장/혼조장에서 어떤 전략이 맞았는가 |

자동학습은 이 5가지를 매일 누적해서 **전략 메모리**로 만드는 과정입니다.

---

## 2. 자동학습 구조는 이렇게 가야 합니다

가장 안전한 구조는 아래입니다.

```text
장중 자동매매
→ 모든 판단 순간 저장
→ 장마감 후 거래/미거래 결과 라벨링
→ 전략별 성과 분석
→ 좋은 패턴/나쁜 패턴 추출
→ AI 복기
→ 다음날 RulePack 개선안 생성
→ 검증 통과 시 적용
```

여기서 중요한 점은 **거래한 종목만 저장하면 부족하다**는 것입니다.

왜냐하면 학습하려면 이런 것도 알아야 하기 때문입니다.

> 매수조건 직전까지 갔지만 안 산 종목이 이후 급등했는가?  
> 시스템이 차단한 종목이 실제로는 좋은 기회였는가?  
> AI 신뢰도는 낮았지만 기술 조건이 완벽했던 종목은 어떻게 됐는가?

그래서 자동학습에는 **체결된 거래 + 미체결 후보 + 차단된 후보**가 모두 필요합니다.

---

## 3. 반드시 저장해야 하는 데이터

자동학습을 하려면 “결과”보다 “판단 당시의 상태”를 저장해야 합니다.

---

### 3.1 진입 판단 시점 Snapshot

매수 여부와 관계없이 후보 종목 15개에 대해 주기적으로 저장해야 합니다.

| **항목** | **예시** |
|---|---|
| 시각 | 09:12:31 |
| 종목코드 | 005930 |
| 현재가 | 72,500 |
| VWAP 위치 | 위 |
| 거래량 배율 | 2.3 |
| 20일선 위치 | 위 |
| RSI | 58.2 |
| 스프레드 | 0.14% |
| 지수 동조 | 충족 |
| AI 신뢰도 | 0.71 |
| 테마 점수 | 0.68 |
| 기술 점수 | 0.82 |
| 기초 점수 | 0.61 |
| 상태 | 매수조건 충족 / 대기 / 차단 |
| 실제 주문 여부 | 주문함 / 안 함 / 차단 |

이 데이터가 있어야 나중에 “그때 왜 샀는지, 왜 안 샀는지”를 복기할 수 있습니다.

---

### 3.2 거래 결과 데이터

체결된 거래는 아래 데이터를 저장해야 합니다.

| **항목** | **설명** |
|---|---|
| 진입 시각 | 매수 체결 시간 |
| 진입 가격 | 평균 체결가 |
| 진입 수량 | 체결 수량 |
| 청산 시각 | 매도 체결 시간 |
| 청산 가격 | 평균 청산가 |
| 보유 시간 | 몇 분 보유했는지 |
| 실현 손익률 | 실제 결과 |
| 최대 유리 변동 | 진입 후 최고 수익률 |
| 최대 불리 변동 | 진입 후 최대 손실률 |
| 청산 사유 | 손절/익절/트레일링/시간손절 |
| 슬리피지 | 기대가 대비 체결가 차이 |
| 룰 위반 여부 | 시스템이 룰을 지켰는지 |

여기서 특히 중요한 것은 **최대 유리 변동**과 **최대 불리 변동**입니다.

예를 들어 어떤 거래가 최종적으로 +0.2%로 끝났더라도, 중간에 +2.8%까지 갔다가 내려온 것이라면 청산 룰이 나빴을 수 있습니다.

반대로 -0.5% 손실로 끝났더라도, 진입 직후 한 번도 +0.2% 이상 가지 못했다면 진입 타이밍이 나빴을 수 있습니다.

---

### 3.3 미진입 후보 데이터

자동학습에서 매우 중요한 부분입니다.

| **항목** | **설명** |
|---|---|
| 후보 종목 | Layer 2 통과 종목 |
| 미진입 사유 | RSI 초과, 스프레드 초과, AI 신뢰도 미달 등 |
| 이후 5분 수익률 | 안 샀는데 이후 어떻게 됐는지 |
| 이후 10분 수익률 | 단타 기준 후속 움직임 |
| 이후 30분 수익률 | 시간손절 기준 비교 |
| 최대 상승률 | 놓친 기회인지 판단 |
| 최대 하락률 | 차단이 적절했는지 판단 |

이 데이터가 있어야 시스템이 이런 걸 배웁니다.

```text
RSI 72라서 안 샀는데 이후 계속 상승했다.
→ RSI 상단 70이 너무 보수적일 수 있음.

스프레드 0.42%라서 차단했는데 이후 급락했다.
→ 스프레드 차단은 유효했음.

AI 신뢰도 0.62라서 안 샀는데 이후 크게 상승했다.
→ AI 신뢰도 기준 0.65가 지나치게 높을 수 있음.
```

## 4. 좋은 전략과 나쁜 전략을 어떻게 구분할까?

단순 수익률만 보면 안 됩니다.  
전략 평가는 여러 지표로 해야 합니다.

---

### 4.1 전략 평가 지표

| **지표** | **의미** |
|---|---|
| 총손익률 | 전체 수익 |
| 승률 | 이긴 거래 비율 |
| 평균 수익 | 수익 거래 평균 |
| 평균 손실 | 손실 거래 평균 |
| 손익비 | 평균 수익 / 평균 손실 |
| MDD | 최대 낙폭 |
| 시간대별 성과 | 오전/오후 성과 차이 |
| 진입 후 최대 상승률 | 진입 품질 |
| 진입 후 최대 하락률 | 진입 위험도 |
| 청산 효율 | 최고 수익 대비 실제 수익 |
| 룰 준수율 | 시스템이 룰대로 실행했는지 |
| 차단 적중률 | 차단한 거래가 실제로 나빴는지 |

---

### 4.2 좋은 전략의 조건

좋은 전략은 단순히 수익이 난 전략이 아닙니다.

좋은 전략은 아래 조건을 만족해야 합니다.

```text
수익이 난다
손실이 제한된다
반복 가능하다
특정 하루 운에 의존하지 않는다
시장 국면별 성과가 설명 가능하다
룰 위반 없이 작동한다
```

예를 들면 이런 전략이 좋습니다.

```text
코스닥 강세 + 거래량 2.5배 이상 + VWAP 위 + 스프레드 0.2% 이하
→ 오전 09:10~10:20 구간에서 승률 64%, 손익비 1.8
```

이런 식으로 **조건 조합 + 시간대 + 시장 국면**을 같이 기억해야 합니다.

---

### 4.3 나쁜 전략의 조건

나쁜 전략은 아래처럼 정의할 수 있습니다.

```text
승률은 높지만 한 번 손실이 크다
수익보다 시간손절이 많다
특정 시간대에서 계속 실패한다
거래량은 많지만 추세 지속이 안 된다
AI 테마 점수는 높지만 가격 반응이 약하다
슬리피지가 커서 기대 수익이 사라진다
```

예:

```text
RSI 68~70 구간에서 진입한 거래
→ 승률 35%, 시간손절 48%, 평균 보유 27분
→ 과열 구간 추격 진입으로 분류
```

이런 패턴은 다음날 RulePack에서 자동으로 약화해야 합니다.

---

## 5. 자동학습은 4단계로 나누는 게 좋습니다

처음부터 머신러닝 모델을 붙이기보다, 아래 4단계로 가는 게 안전합니다.

---

### Level 1. 통계 기반 학습

가장 먼저 해야 할 방식입니다.

```text
조건별 성과 집계
시간대별 성과 집계
시장 국면별 성과 집계
전략별 승률/손익비 계산
차단 사유별 사후 성과 계산
```

예:

| **조건** | **거래수** | **승률** | **평균손익** | **판단** |
|---|---:|---:|---:|---|
| 거래량 2.0~2.5배 | 31 | 51% | +0.12% | 유지 |
| 거래량 2.5~3.5배 | 26 | 63% | +0.41% | 강화 |
| RSI 40~55 | 18 | 61% | +0.32% | 유지 |
| RSI 65~70 | 22 | 38% | -0.18% | 약화 |
| 스프레드 0.3% 초과 | 11 | 27% | -0.44% | 차단 유지 |

이 단계만 제대로 해도 자동매매 품질이 크게 좋아집니다.

---

### Level 2. 룰 개선 제안

통계 결과를 바탕으로 시스템이 다음날 룰 개선안을 만듭니다.

예:

```json
{
  "suggestion_id": "rule_adjust_20260429_001",
  "target_rule": "rsi_max",
  "current_value": 70,
  "suggested_value": 67,
  "reason": "최근 20거래 기준 RSI 67~70 구간의 시간손절 비율이 45%로 높음",
  "confidence": 0.74,
  "impact": "진입 횟수 감소, 손실 거래 감소 기대"
}
```

이 단계에서는 시스템이 직접 실전 룰을 바꾸지 않고, **개선안을 생성**합니다.

---

### Level 3. 검증 후 자동 반영

충분한 데이터가 쌓이면, 특정 범위 안에서는 자동 반영할 수 있습니다.

예를 들어 다음 같은 안전장치를 둡니다.

```text
RSI 범위는 하루에 최대 3포인트만 조정 가능
거래량 배율은 0.2 단위로만 조정 가능
손절폭은 절대 완화 불가
일일 손실한도는 AI가 변경 불가
포지션 한도는 AI가 증가 불가
```

즉, AI가 개선은 해도 **리스크 룰은 건드리지 못하게** 해야 합니다.

---

### Level 4. 모델 기반 학습

나중에는 ML 모델을 붙일 수 있습니다.

예:

| **모델** | **역할** |
|---|---|
| 진입 품질 모델 | 현재 조건에서 10분 내 유리하게 움직일 확률 |
| 청산 최적화 모델 | 익절/트레일링/시간손절 중 어느 쪽이 나은지 |
| 차단 판단 모델 | 지금 주문을 차단하는 게 맞는지 |
| 시장 국면 분류 모델 | 오늘이 추세장/박스장/급변동장인지 |

하지만 이건 후순위입니다.  
처음에는 **통계 기반 학습 + AI 복기 + RulePack 개선안**이 가장 안정적입니다.

---

## 6. “좋은 타이밍”은 어떻게 기억해야 하나?

좋은 타이밍은 단순히 수익 난 순간이 아닙니다.  
진입 후 바로 유리하게 움직이고, 불리한 흔들림이 작고, 청산 룰과 잘 맞는 진입이 좋은 타이밍입니다.

---

### 6.1 좋은 진입 타이밍 정의

좋은 진입은 예를 들어 아래처럼 정의할 수 있습니다.

```text
진입 후 5분 내 +0.5% 이상 도달
진입 후 최대 역행이 -0.4% 이하
10분 내 VWAP 위 유지
거래량 증가 지속
호가 스프레드 안정
```

이걸 점수화하면 됩니다.

예:

```text
Entry Quality Score
- 진입 후 5분 수익률
- 최대 불리 변동
- 목표가 도달 여부
- 시간손절 회피 여부
- 슬리피지
```

수식으로 표현하면 이런 식입니다.

$$
EntryScore = Return_{5m} + 0.5 \times Return_{10m} - MAE - Slippage
$$

여기서:

| **항목** | **의미** |
|---|---|
| $$Return_{5m}$$ | 진입 후 5분 수익률 |
| $$Return_{10m}$$ | 진입 후 10분 수익률 |
| $$MAE$$ | 최대 불리 변동 |
| $$Slippage$$ | 체결 불리 비용 |

이 점수가 높은 조건 조합을 “좋은 타이밍 패턴”으로 저장합니다.

---

### 6.2 나쁜 진입 타이밍 정의

나쁜 진입은 아래처럼 정의할 수 있습니다.

```text
진입 후 5분 내 -0.5% 이상 역행
30분 내 +0.5% 미달성
시간손절 발생
스프레드 확대
거래량 급감
지수 방향과 역행
```

예:

```text
Bad Timing Pattern:
- RSI 68 이상
- 거래량 배율은 높지만 3분 연속 감소
- VWAP 위지만 지수 하락 전환
- 스프레드 0.28% 이상
→ 시간손절 확률 높음
```

이런 패턴은 다음날 RulePack에서 감점 요인으로 들어가야 합니다.

---

## 7. 전략 메모리 구조

자동학습을 하려면 “메모리”가 필요합니다.  
단순 로그가 아니라 학습 가능한 형태로 저장해야 합니다.

---

### 7.1 추천 DB 테이블 구조

```text
trade_events
- 모든 주문/체결/청산 이벤트

decision_snapshots
- 매수/미매수/차단 판단 당시 조건

strategy_results
- 전략별 일일 성과

pattern_memory
- 좋은/나쁜 조건 조합 저장

rule_suggestions
- 다음 RulePack 개선 제안

market_regime_daily
- 오늘 시장 국면 기록

ai_review_reports
- AI 복기 리포트
```
