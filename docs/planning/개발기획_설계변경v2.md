## 1. 전체 아키텍처 변경

### Before
```
S5 → AI가 RulePack 전체 생성 (매일 새 룰)
     ↓
모든 종목에 동일 RulePack 적용
```

### After
```
고정 Base RulePack (base-v1.0)
     +
고정 Risk Profile Pack (profile-v1.x, 4종)
     +
S5 Daily Trading Plan (매일 생성 — 어떤 종목에 어떤 프로필 배정)
     ↓
resolve_symbol_rule() → 종목별 최종 룰 계산
     ↓
메모리 캐시 (09:00 로드 → 장마감 초기화)
     ↓
WS tick → Decision Engine → Risk Guard → KIS 주문
```

### Rule 적용 우선순위 (불변)
```
1. Emergency Halt
2. Global Risk Guard
3. 장마감 강제청산 정책
4. Symbol Override
5. Risk Profile
6. Base RulePack
7. Daily Trading Plan
8. AI Suggestion  ← 가장 낮음
```

## 2. DB 스키마 변경

### 2.1 신규 테이블 (추가)

```sql
-- Base RulePack (고정, 버전 관리)
CREATE TABLE IF NOT EXISTS base_rulepacks (
    id              TEXT PRIMARY KEY,           -- "base-v1.0"
    version         TEXT NOT NULL,
    take_profit_enabled     INTEGER NOT NULL DEFAULT 0,
    force_daily_close       INTEGER NOT NULL DEFAULT 1,
    force_exit_time         TEXT NOT NULL DEFAULT "15:20:00",
    stop_price_can_only_increase INTEGER NOT NULL DEFAULT 1,
    order_execution TEXT NOT NULL DEFAULT '{}', -- JSON
    created_at      TEXT NOT NULL,
    is_active       INTEGER NOT NULL DEFAULT 1
);

-- Risk Profile Pack (4종 세트, 버전 관리)
CREATE TABLE IF NOT EXISTS risk_profile_packs (
    id              TEXT PRIMARY KEY,           -- "profile-v1.0"
    version         TEXT NOT NULL,
    profiles        TEXT NOT NULL,              -- JSON: {LOW_VOL, MID_VOL, HIGH_VOL, THEME_SPIKE}
    created_at      TEXT NOT NULL,
    is_active       INTEGER NOT NULL DEFAULT 1
);

-- Daily Trading Plan (매일 생성)
CREATE TABLE IF NOT EXISTS daily_trading_plans (
    id              TEXT PRIMARY KEY,           -- "daily-2026-05-03"
    trade_date      TEXT NOT NULL UNIQUE,       -- "2026-05-03"
    market_tone     TEXT NOT NULL DEFAULT 'neutral',
    trading_intensity TEXT NOT NULL DEFAULT 'normal',
    base_rulepack_id TEXT NOT NULL,
    risk_profile_pack_id TEXT NOT NULL,
    new_entry_allowed INTEGER NOT NULL DEFAULT 1,
    daily_overrides TEXT NOT NULL DEFAULT '{}', -- JSON
    symbol_assignments TEXT NOT NULL DEFAULT '[]', -- JSON
    excluded_symbols  TEXT NOT NULL DEFAULT '[]',  -- JSON
    llm_summary     TEXT NOT NULL DEFAULT '',
    provider        TEXT NOT NULL DEFAULT '',
    status          TEXT NOT NULL DEFAULT 'draft', -- draft|validated|active|archived
    validation_result TEXT NOT NULL DEFAULT '{}',  -- JSON
    created_at      TEXT NOT NULL,
    activated_at    TEXT
);
CREATE INDEX IF NOT EXISTS idx_daily_plan_date ON daily_trading_plans(trade_date);

-- Symbol Overrides (종목별 프로필 값 override)
CREATE TABLE IF NOT EXISTS symbol_overrides (
    id              TEXT PRIMARY KEY,
    symbol_code     TEXT NOT NULL UNIQUE,
    symbol_name     TEXT NOT NULL DEFAULT '',
    default_profile TEXT NOT NULL DEFAULT 'MID_VOL',
    override_values TEXT NOT NULL DEFAULT '{}',  -- JSON: 덮어쓸 필드만
    is_active       INTEGER NOT NULL DEFAULT 1,
    created_at      TEXT NOT NULL,
    updated_at      TEXT NOT NULL
);

-- Rule Composition (종목별 최종 룰 계산 결과 캐시/이력)
CREATE TABLE IF NOT EXISTS rule_compositions (
    id              TEXT PRIMARY KEY,
    trade_date      TEXT NOT NULL,
    symbol_code     TEXT NOT NULL,
    final_rule      TEXT NOT NULL,              -- JSON: resolve_symbol_rule() 결과
    base_rulepack_id TEXT NOT NULL,
    risk_profile_pack_id TEXT NOT NULL,
    daily_plan_id   TEXT NOT NULL,
    profile_assigned TEXT NOT NULL,
    created_at      TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_rule_comp_date ON rule_compositions(trade_date);

-- Position Stop States (트레일링 스탑 실시간 상태)
CREATE TABLE IF NOT EXISTS position_stop_states (
    position_id     TEXT PRIMARY KEY,
    symbol_code     TEXT NOT NULL,
    entry_price     REAL NOT NULL DEFAULT 0.0,
    highest_price_since_entry REAL NOT NULL DEFAULT 0.0,
    initial_stop_price REAL NOT NULL DEFAULT 0.0,
    trailing_stop_price REAL NOT NULL DEFAULT 0.0,
    active_stop_price REAL NOT NULL DEFAULT 0.0,
    trailing_active INTEGER NOT NULL DEFAULT 0,
    profile_assigned TEXT NOT NULL DEFAULT 'MID_VOL',
    last_updated_at TEXT NOT NULL
);
```

## 5. 핵심 엔진 로직

### 5.1 resolve_symbol_rule()

`backend/services/engine/rule_resolver.py` (신규):

```python
def resolve_symbol_rule(
    symbol_code: str,
    base_rulepack: dict,
    profile_pack: dict,
    daily_plan: dict,
    symbol_overrides: dict,
    global_risk: dict,
) -> dict:
    # 1. Daily Plan에서 배정된 프로필 조회
    assignments = {a["code"]: a for a in daily_plan.get("symbol_assignments", [])}
    assignment = assignments.get(symbol_code, {})
    profile_name = assignment.get("profile", "MID_VOL")

    # 2. 레이어별 룰 병합 (낮은 우선순위 → 높은 우선순위 순)
    final_rule = {}
    final_rule.update(base_rulepack)                                    # 6순위
    final_rule.update(profile_pack["profiles"][profile_name])           # 5순위
    final_rule.update(symbol_overrides.get(symbol_code, {}))           # 4순위

    # 3. Global Risk Guard 강제 적용 (절대 완화 불가)
    final_rule["max_position_rate"] = min(
        final_rule.get("max_position_rate", global_risk["max_position_rate_per_stock"]),
        global_risk["max_position_rate_per_stock"],
    )
    final_rule["force_exit_time"] = global_risk["force_exit_time"]
    final_rule["take_profit_enabled"] = False               # 항상 OFF
    final_rule["stop_price_can_only_increase"] = True       # 항상 ON
    final_rule["profile_assigned"] = profile_name

    return final_rule
```
