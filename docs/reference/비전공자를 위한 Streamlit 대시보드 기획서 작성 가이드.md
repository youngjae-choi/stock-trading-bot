비전공자도 AI(Cursor 등)에게 정확하게 지시하여 원하는 화면을 만들어낼 수 있는 Streamlit 대시보드 화면 기획서 작성 가이드를 정리해 드립니다.
Streamlit은 복잡한 HTML/CSS 디자인 없이 파이썬 코드의 순서대로 화면이 위에서 아래로 그려지는 특징이 있습니다. 따라서 기획서를 작성할 때는 복잡한 그림 가이드보다 '어떤 설정'으로, '어떤 상자(Container)' 안에, '몇 개의 칸(Column)'을 나눌지 텍스트로 명확히 정의해 주는 것이 가장 효과적입니다
.
--------------------------------------------------------------------------------
💡 Streamlit 화면 기획서 작성 5단계 가이드
1단계: 전체 페이지 기본 설정 (Page Config)
대시보드의 뼈대를 잡는 가장 기초적인 설정입니다.
레이아웃 설정: 화면을 넓게 쓰기 위해 layout="wide" 설정을 반드시 기획서에 포함해야 합니다. 이 설정을 하지 않으면 화면의 3/4 정도만 채워져 대시보드가 답답해 보일 수 있습니다
.
타이틀 및 아이콘: 웹 브라우저 탭에 표시될 페이지 제목(page_title)과 아이콘(page_icon)을 지정합니다
.
2단계: 헤더(Header) 및 구역(Container) 나누기
전체 화면을 논리적인 덩어리(Container)로 나눕니다.
헤더: 대시보드 최상단에 표시될 큰 제목(예: "트레이딩 터미널")을 정의합니다
.
컨테이너(Container): 연관된 기능끼리 묶어줄 상자를 기획합니다. Streamlit은 st.container()를 사용하여 임의의 크기로 구역을 나눌 수 있으며, 컨테이너 안에 또 다른 컨테이너를 중첩(Nested)해서 구성할 수도 있습니다
.
3단계: 다단 배치 (Columns)
컨테이너 내부의 요소를 가로로 나란히 배치하기 위해 단(Column)을 나눕니다.
예를 들어, 상단 제어 패널에 '설정 파일 사용 여부', '알림 수신처', '트레이딩 플랫폼', '매매 실행 여부'를 나란히 배치하고 싶다면 **"해당 컨테이너를 4개의 컬럼으로 분할해달라"**고 명시합니다
.
4단계: 위젯(Widgets) 상세 정의
각 컬럼에 들어갈 버튼이나 선택 박스의 세부 동작을 정의합니다.
드롭다운 메뉴 (Selectbox): 제목, 선택지(Options), 빈값 설정(index=None), 안내 문구(placeholder: "옵션을 선택하세요" 등)를 명확히 기재합니다
.
버튼 (Button): 시작/정지 버튼의 경우 기본값을 False로 두고, 사용자가 클릭했을 때 True로 바뀌며 특정 로직이 실행되도록 기획합니다
.
5단계: 조건부 로직 (Branching Logic) 기획
사용자의 선택에 따라 화면이 동적으로 변하는 분기(Branching) 조건을 명시합니다.
예를 들어, 알림 설정에서 'Discord'를 선택하면 화면에 "Discord 연결 중..."이라는 텍스트(st.write)를 띄우거나
, 설정 파일을 사용하지 않음('No')으로 선택하면 사용자에게 API 키를 직접 입력받는 입력창을 띄우도록 지시합니다
.

📝 실전 활용용 텍스트 화면 기획서 템플릿
위의 가이드를 바탕으로 AI에게 전달할 수 있는 완벽한 텍스트 기획서 템플릿을 제공해 드립니다. 이 양식을 복사하여 필요한 내용만 수정해 Cursor AI에게 전달해 보세요.
[대시보드 기획서] 너는 파이썬 Streamlit 전문가야. 아래의 기획안을 바탕으로 대시보드 UI 코드를 작성해 줘.
1. 기본 설정 (Page Config)
페이지 타이틀: "나만의 트레이딩 봇"
페이지 아이콘: "🤖"
레이아웃: wide 모드로 설정해서 화면 전체를 채워줘
.
2. 헤더 및 상단 컨트롤 컨테이너 (Header Container)
최상단에 "📈 통합 자동매매 터미널" 이라는 헤더를 만들어줘
.
그 아래에 메인 컨트롤을 위한 st.container()를 하나 생성해줘
.
이 컨테이너 내부는 st.columns(4)를 써서 4개의 가로 컬럼으로 분할해줘
.
3. 상단 4개 컬럼 상세 구성
컬럼 1 (설정 파일): st.selectbox를 사용. 옵션은 ["Yes", "No"], 기본 선택값 없음(index=None), placeholder는 "설정 파일 사용 여부"
.
컬럼 2 (알림 리스너): st.selectbox를 사용. 옵션은 ["Discord", "Direct"], placeholder는 "알림 채널 선택"
.
컬럼 3 (트레이딩 플랫폼): st.selectbox를 사용. 옵션은 ["한국투자증권", "모의투자", "Alpaca"], placeholder는 "플랫폼 선택"
.
컬럼 4 (매매 실행 여부): st.selectbox를 사용. 옵션은 ["Yes", "No"]
.
4. 조건부 로직 (Branching Logic) 적용
컬럼 2에서 "Discord"를 선택한 경우, 하단에 "Discord 연결 중..."이라는 텍스트 메시지를 띄우고 토큰을 입력받는 입력창을 조건부로 띄워줘
.
컬럼 1에서 설정 파일을 사용하지 않는다("No")고 선택한 경우, 사용자가 직접 API 키를 입력할 수 있는 텍스트 박스를 띄워줘
.
5. 하단 실행 영역 (Action Container)
컨트롤 컨테이너 아래에 새로운 컨테이너를 만들고, 봇 구동을 위한 Start/Stop 버튼을 만들어줘. 버튼의 기본 상태는 False이며 누르면 True로 전환되게 해줘
.
이러한 형태의 기획서를 제공하면, AI는 각 컨테이너와 컬럼의 부모-자식 관계를 완벽하게 이해하고 PC와 모바일 모두에서 깔끔하게 떨어지는 반응형 대시보드 코드를 한 번에 짜줄 것입니다.
