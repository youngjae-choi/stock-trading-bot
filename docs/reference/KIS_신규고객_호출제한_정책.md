## 1. 정책 개요

한국투자증권 Open API(KIS API)는 시스템 안정성 및 공정한 자원 배분을 위해 **신규 API 서비스 신청 고객**을 대상으로 초기 호출 유량을 제한하고, 일정 기간 경과 후 자동으로 상향하는 정책을 시행한다.

> ⚠️ **중요: 이 정책은 실전(Real) 환경에만 적용되며, 모의투자 환경은 대상에서 제외된다.**

## 3. 상세 제한 정책

신규 고객은 서비스 신청 직후 일정 기간 동안 낮은 유량 제한이 적용되며, 이후 단계적으로 상향된다.

### 3-1. 기간별 호출 유량 (REST API)

| 구분 | 적용 기간 | 호출 제한 (TPS) | 비고 |
|---|---|---|---|
| **신규 신청 단계** | **신청일 포함 3일간** | **초당 3건 (3 TPS)** | 초기 안정성 확보 기간 |
| **기본 유량 단계** | 신청 3일 경과 후 | **초당 20건 (20 TPS)** | 표준 기본 유량으로 자동 상향 |
| **갱신 시** | 서비스 갱신 시 | **자동 상향 적용** | 별도 신청 없이 갱신 시점에 반영 |

* ※ 위 제한은 AppKey 단위가 아닌 계정/IP 등의 복합 기준으로 적용될 수 있다.
* ※ 초당 거래건수 초과 시 `EGW00201` 오류가 반환된다.

### 3-2. 정책 특징

1. **자동 상향**: 신청 후 3일이 경과하면 시스템에서 자동으로 기본 유량(20 TPS)으로 전환된다.
2. **갱신 연동**: 서비스 유효기간 만료 후 갱신 시에도 상향된 정책이 유지되거나 자동 적용된다.
3. **환경 분리**: 모의투자 서버는 본 정책의 영향을 받지 않으므로, 개발 및 테스트는 모의투자 환경을 활용하는 것이 권장된다.

## 4. 대응 및 운영 가이드

### 4-1. EGW00201 (초당 거래건수 초과) 대응

신규 3일 구간에서 빈번하게 발생할 수 있으며, 아래와 같이 대응한다.

* **RateLimiter 설정**: 신규 계정의 경우 내부 RateLimiter를 **2 TPS** 정도로 보수적으로 설정하여 여유를 둔다.
* **지연 및 재시도**: 오류 발생 시 최소 1초 이상의 대기(Sleep) 후 재시도를 수행한다.
* **모니터링**: `logs/` 내의 API 호출 로그를 통해 `EGW00201` 발생 빈도를 추적한다.

### 4-2. 운영 체크리스트

- [ ] **계좌 개설/서비스 신청일 확인**: 현재가 신청 후 며칠째인지 확인 (3일 경과 여부)
- [ ] **환경 확인**: `.env`의 `KIS_CAN_MOCK` 또는 URL 설정이 실전인지 모의투자 인지 확인
- [ ] **유량 조정**: 신규 3일 이내라면 Oracle을 통해 코드의 `max_calls` 설정을 하향 조정 요청
- [ ] **상향 확인**: 3일 경과 후 호출 테스트를 통해 3 TPS 이상의 속도에서도 정상 작동하는지 검증

### 4-3. 현재 저장소 반영 기준

운영 구현은 아래 기준으로 반영한다.

- `KIS_SERVICE_APPLY_DATE=YYYY-MM-DD`를 설정하면 신청일 포함 3일 동안 신규 계정 프로필을 적용한다.
- 신규 실전 프로필의 내부 `RateLimiter` 기본값은 **2 TPS**로 둬서 3 TPS 제한보다 보수적으로 운영한다.
- 3일 경과 후에는 실전 기본 프로필을 **20 TPS**로 자동 상향한다.
- `KIS_RATE_LIMIT_PROFILE=auto|new_account|standard`로 수동 강제 전환이 가능하다.
- `KIS_RATE_LIMIT_RPS`를 지정하면 위 프로필보다 우선하는 운영자 고정값으로 동작한다.
- 헬스 체크 응답(`/health`)에서 현재 적용 중인 프로필과 RPS를 확인할 수 있다.
