# KIS 필터 호출체인 개발 우선순위 (2026-04-22)

## 1) 결론
- 조회 필터 엔진은 `종목별 단건 조회` 중심이 아니라, `유니버스 일괄조회 API`를 1차로 사용해야 효율적이다.
- KIS 문서 기준 국내/해외 조회 REST 140개 중, 유니버스 직접 생성에 유효한 API(`universe_direct`)가 32개 존재한다.
- 다만 모의 지원은 매우 제한적이다. 140개 중 모의지원 `17`, 미지원 `123`.

## 2) 필터 지원 판정 (핵심)

### A. 직접 지원 (유니버스 API로 즉시 후보 추출)
- 시가총액 상위: `/uapi/domestic-stock/v1/ranking/market-cap`, `/uapi/overseas-stock/v1/ranking/market-cap`
- 거래량 순위: `/uapi/domestic-stock/v1/quotations/volume-rank`, `/uapi/overseas-stock/v1/ranking/trade-vol`
- 체결강도 상위: `/uapi/domestic-stock/v1/ranking/volume-power`, `/uapi/overseas-stock/v1/ranking/volume-power`
- 조건검색 목록/결과: `/uapi/domestic-stock/v1/quotations/psearch-title`, `/uapi/domestic-stock/v1/quotations/psearch-result`
- 등락률/시장가치/이격도/호가잔량/시간외 등 각종 랭킹: `domestic-stock/v1/ranking/*`

### B. 부분 지원 (팩터 API + 후보군 조합)
- 시장별 투자자매매동향(일/시세):
  - `/uapi/domestic-stock/v1/quotations/inquire-investor-daily-by-market`
  - `/uapi/domestic-stock/v1/quotations/inquire-investor-time-by-market`
- 종목별 수급/외인기관:
  - `/uapi/domestic-stock/v1/quotations/investor-trade-by-stock-daily`
  - `/uapi/domestic-stock/v1/quotations/investor-trend-estimate`
  - `/uapi/domestic-stock/v1/quotations/foreign-institution-total`

### C. 계산 필요 (차트/시계열로 후처리)
- 국내: `/uapi/domestic-stock/v1/quotations/inquire-daily-itemchartprice`, `/inquire-time-itemchartprice`
- 해외: `/uapi/overseas-price/v1/quotations/dailyprice`, `/inquire-time-itemchartprice`
- 대상 지표: RSI, 이동평균, 거래량 평균/증가율, 변동성, 돌파 신호

### D. 단건 보강 (최종 후보 소수에만 사용)
- 현재가/호가/체결:
  - `/uapi/domestic-stock/v1/quotations/inquire-price`
  - `/uapi/domestic-stock/v1/quotations/inquire-asking-price-exp-ccn`
  - `/uapi/domestic-stock/v1/quotations/inquire-ccnl`

## 3) 최적 호출 체인 (권장 아키텍처)

### 체인 1: 랭킹 기반 (가장 효율적, 기본)
1. 유니버스 생성: `ranking/*` 또는 `volume-rank`로 Top N 추출
2. 팩터 결합: 투자자/외인기관 API를 후보군에 조인
3. 계산 지표: 필요한 종목만 차트 API 호출 후 RSI/MA 계산
4. 최종 보강: 통과 종목에만 `inquire-price`/호가 조회

### 체인 2: 조건검색 기반 (사용자 전략 직접 반영)
1. 조건 목록 조회: `psearch-title`
2. 조건 결과 조회: `psearch-result`
3. 후속 필터(수급/기술지표) 적용
4. 단건 보강 후 결과 출력/주문 후보 확정

### 체인 3: 수급 기반 스캐너
1. 시장/종목 투자자 동향 API로 1차 후보
2. 외인기관 추정가집계 조인
3. 가격/체결강도 랭킹 결합
4. 최종 후보에만 단건 보강

### 체인 4: 기술지표 기반 스캐너
1. 유니버스는 랭킹 Top N으로 축소
2. 차트 API로 RSI/MA/거래량 증가율 계산
3. 조건 AND 필터
4. 통과 종목 단건 보강

## 4) 개발 우선순위

### P0 (즉시)
- 현재 엔진의 `유니버스 생성`을 stock_master 순회가 아닌 랭킹/조건검색 기반으로 교체
- 지원 API 우선순위
  - 국내 `volume-rank`, `ranking/market-cap`, `ranking/volume-power`
  - 조건검색 `psearch-title`, `psearch-result`
- 결과 스키마에 `universe_source_api`, `api_call_count`, `api_fail_count` 표준화

### P1
- 수급 계열 조인
  - `inquire-investor-time-by-market`, `investor-trend-estimate`, `foreign-institution-total`
- 계산지표 파이프라인 분리(차트 호출 배치 + 캐시)

### P2
- 해외주식 동일 구조 확장
  - `ranking/trade-vol`, `ranking/market-cap`, `ranking/volume-power`
  - 해외 차트/기본시세 결합
- 실시간(웹소켓) 이벤트 기반 트리거 고도화

## 5) 성능/안정성 규칙
- 단건 조회는 후보 축소 후에만 허용 (예: Top 20~50)
- 동일 종목 중복조회 금지(캐시/배치)
- API 제한 대응: 동시성 제한 + 지수 백오프 + 부분실패 허용
- 모의 미지원 API는 테스트 UI에 명시하고 실전토큰에서만 실행

## 6) 현재 상태 대비 변경 포인트
- 기존: `유니버스 후보 -> 종목 단건 조회 반복` 중심
- 목표: `유니버스 API 우선 -> 조인/계산 -> 최소 단건보강`
