## 배경

Backend가 장중 재선별 시스템에 다음 3가지 신기능을 추가합니다:
1. 점심 이후 슬롯 (13:00, 14:00)
2. 섹터 회전 감지
3. 포지션 교체 신호 (B 방식 — 신호만, 강제 교체 없음)

이 신기능들의 **결과를 PM이 한눈에 확인할 수 있는 시각화**가 필요합니다.

## 의존 API (Codex 작업 후 사용 가능)

### API 1: 재선별 실행 통계
```
GET /api/v1/trading-monitor/reselection-stats?trade_date=YYYY-MM-DD
응답:
{
  "ok": true,
  "payload": {
    "trade_date": "2026-05-25",
    "slots": [
      {"slot": "09:30", "triggered": true, "reason": "...", "new_candidates": 12},
      {"slot": "10:30", "triggered": false, "reason": "noise threshold"},
      {"slot": "11:30", "triggered": true, ...},
      {"slot": "13:00", "triggered": false, ...},
      {"slot": "14:00", "triggered": true, ...}
    ],
    "sector_rotations": [
      {"slot": "14:00", "top_sectors": ["반도체", "2차전지"], "bottom_sectors": ["바이오"], "gap_pct": 5.4, "triggered": true}
    ],
    "replacement_signals": [...]
  }
}
```

### API 2: 교체 신호 조회
```
GET /api/v1/trading-monitor/replacement-signals?trade_date=YYYY-MM-DD
응답:
{
  "ok": true,
  "payload": {
    "signals": [
      {
        "id": 1,
        "slot": "10:30",
        "current": {"symbol": "005930", "name": "삼성전자", "score": 0.65, "pnl_pct": -1.2},
        "new": {"symbol": "035420", "name": "NAVER", "score": 0.85},
        "score_gap": 30.7,
        "reason": "거래량 급증 + 모멘텀 상위",
        "created_at": "2026-05-25T10:30:15"
      }
    ]
  }
}
```

### API 3: System Settings (Kill Switch 토글용)
이미 존재하는 settings API 활용. 토글 대상 키 4개:
- `intraday_refresh.master_enabled`
- `intraday_refresh.lunch_slots_enabled`
- `intraday_refresh.sector_rotation_enabled`
- `intraday_refresh.replacement_signal_enabled`

## 작업 범위

### Task 1: 재선별 타임라인 카드 ("오늘" 화면)

"오늘" 또는 "Today" 화면에 신규 카드 추가:

```
┌─────────────────────────────────────────────────┐
│ 📅 장중 재선별 타임라인 (2026-05-25)              │
├─────────────────────────────────────────────────┤
│ 09:30  ✅ 트리거됨  defensive +2.5%  (12종목)    │
│ 10:30  ⏭️ 스킵     평균 +0.3%                   │
│ 11:30  ✅ 트리거됨  섹터 회전 감지 (8종목)        │
│ 13:00  ⏭️ 스킵                                 │
│ 14:00  ✅ 트리거됨  defensive +2.1%  (10종목)    │
└─────────────────────────────────────────────────┘
```

**완료 기준**:
- [ ] 5개 슬롯 모두 표시 (실행 전/후 상태 구분)
- [ ] 트리거 시 사유 표시
- [ ] 시간 지나기 전 슬롯은 "대기" 표시
- [ ] 모바일/PC 대응

---

### Task 2: 섹터 회전 표시 (Task 1 카드 내부 또는 별도)

섹터 회전 감지된 슬롯에는 추가 정보:
```
🔄 14:00 섹터 회전
   상위: 반도체(+4.2%), 2차전지(+3.1%)
   하위: 바이오(-1.8%), 통신(-1.2%)
   갭: 5.4%
```

**완료 기준**:
- [ ] 상위 2개, 하위 그룹 표시
- [ ] 갭 % 표시
- [ ] 시각적으로 회전 방향 표현 (화살표 또는 색상)

---

### Task 3: 교체 신호 카드 (별도)

"오늘" 화면 또는 "Trading" 탭에 신규 카드:

```
┌─────────────────────────────────────────────────────────┐
│ 🎯 포지션 교체 신호 (오늘 2건)                            │
├─────────────────────────────────────────────────────────┤
│ ▼ 10:30 신호                                              │
│  📊 보유: 삼성전자 (점수 0.65, 손익 -1.2%)                 │
│  🎯 후보: NAVER (점수 0.85, +30.7%↑)                      │
│  사유: 거래량 급증 + 모멘텀 상위                            │
│                                                          │
│ ▼ 14:00 신호                                              │
│  ...                                                     │
│                                                          │
│ ⚠️ 강제 교체 없음. 트레일링 스탑 발동 시 자연 교체.          │
└─────────────────────────────────────────────────────────┘
```

**완료 기준**:
- [ ] 카드 헤더에 총 신호 수 표시
- [ ] 신호별 expandable/collapsible
- [ ] 보유 종목 / 신규 후보 점수 비교 시각화
- [ ] 사유 텍스트 표시
- [ ] "강제 교체 없음" 안내 문구 (PM이 헷갈리지 않도록)
- [ ] 신호 0건일 때 "오늘 교체 신호 없음" 표시

### Task 4: Kill Switch UI (설정 화면)

"Settings" 화면에 신규 섹션 추가:

```
┌─────────────────────────────────────────────────┐
│ ⚙️ 장중 재선별 기능 활성화 (긴급 비활성화)         │
├─────────────────────────────────────────────────┤
│ 전체 활성화                              [✓]    │  ← master
│   └ 점심 슬롯 (13:00/14:00)             [✓]    │  ← lunch_slots
│   └ 섹터 회전 감지                       [✓]    │  ← sector_rotation
│   └ 교체 신호 발생                       [✓]    │  ← replacement_signal
│                                                  │
│ ⚠️ 끄면 즉시 반영됨 (재시작 불필요)               │
└─────────────────────────────────────────────────┘
```

**완료 기준**:
- [ ] 4개 토글 (master + 3개 sub-feature)
- [ ] master OFF 시 sub-feature 토글 disabled 처리
- [ ] 토글 변경 → settings API 호출 → 성공/실패 토스트
