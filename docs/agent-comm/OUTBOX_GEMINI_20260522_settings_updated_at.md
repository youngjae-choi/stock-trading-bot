# OUTBOX: Settings 화면 항목별 최종 업데이트 시각 표시 완료

**날짜:** 2026-05-22  
**작업자:** Gemini (Frontend)

## 작업 요약
Settings 화면의 주요 설정 항목(리스크 한도, 스케줄러, 청산 규칙, 매수 조건) 옆에 "언제, 누가 마지막으로 수정했는지"를 표시하는 기능을 구현했습니다.

## 변경 사항

### 1. 공통 유틸리티 추가 (`console-diagnostics.js`)
- `loadSettingsMapFull()`: `/api/v1/settings` API를 호출하여 값뿐만 아니라 `updated_at`, `updated_by` 메타데이터를 포함한 맵을 반환합니다.
- `_fmtSettingTs(isoStr, updatedBy)`: ISO 시각을 `MM-DD HH:mm` 형식으로 변환하고 수정자 이름을 결합하여 HTML span으로 반환합니다.

### 2. Settings 화면 구현 업데이트 (`console-settings.js`)
- `_insertSettingTs()`: 일반 input 필드 아래에 타임스탬프를 동적으로 삽입하는 유틸리티를 추가했습니다.
- **리스크 관리 섹션:** 각 설정(손실한도, 최대 포지션 등) 입력란 아래에 최종 수정 시각을 표시합니다.
- **스케줄러 섹션:** 테이블에 '최종 수정' 열을 추가하여 각 단계별 수정 이력을 표시합니다.
- **청산 규칙 섹션:** 테이블 마지막 열에 '최종 수정' 정보를 추가했습니다.
- **매수 조건 섹션:** 가드레일 입력란(input) 아래에 타임스탬프를 표시하도록 업데이트했습니다.

### 3. UI 구조 변경 (`console.html`)
- 스케줄러 및 청산 규칙 테이블에 '최종 수정' 헤더(`<th>`)를 추가했습니다.
- 테이블 로딩 메시지의 `colspan`을 6으로 확장하여 레이아웃 깨짐을 방지했습니다.

## 확인 방법
1. 브라우저 새로고침 후 Settings 화면 접속.
2. 각 입력 항목 아래 또는 테이블 열에 `05-22 14:30 · user` 형태의 텍스트가 표시되는지 확인.
3. 값이 없는 항목은 조용히 빈 칸으로 처리됨을 확인.

**비고:** 서버 재시작 없이 정적 파일 반영만으로 즉시 확인 가능합니다.
