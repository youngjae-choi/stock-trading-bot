# 강세주 탐색 단타 엔진 — 설계서 v0.1 (브레인스토밍 산출)

작성일: 2026-06-06 · 대상: S3 유니버스 / S6 매수 엔진 / 태깅·리뷰·학습 / UI(Trade History·Monitoring·Settings·Review)

## 한 줄 요약
**모의계좌에서 그물을 최대로(모든 조건 OR) 던져 하루 많은 종목을 매매하고, 매수마다 "조건·시장맥락·결과"를 통째로 태깅한다. 쌓인 데이터로 기대값(EV) 낮은 조건을 가지치기해 "20승80패 → 80승20패"로 수렴시킨다.**

## WHY (PM 발화 그대로 인용)
> 모두 OR 조건으로 해보자. 이렇게 과도하게 매수종목을 늘려 승율이 낮은 매수조건을 찾아 추후에 제거하고 싶어. 풀 예수금 사용, 하루에 가급적 많은 종목을 매매해서 올바른 시장상황파악 그에 따른 매수전략, 손절전략을 만들어야해. … 내일은 20승 80패라면 한달뒤 두달뒤에는 80승 20패 … 모의계좌일때 하지 않으면 안돼.
> (UI) Trade history 및 Monitoring 화면에 매수사유·손절사유, 리뷰에서 분석. Setting에 매수조건 설정값, 필요 데이터값을 AND로 그룹핑해 레짐/Risk Profile에 할당.

탐색(explore)→측정→가지치기(prune)→수렴(exploit). 실계좌에선 불가능한 실험을 **모의계좌 샌드박스**에서 수행한다.

## 완성 사이클 — 선정 → 매수 → 결과 (각 단계에 "사유")
> PM: "종목부터 찾아야 하지 않아? … 어떤 종목을 왜 [선정했고] 왜 매수했고 그 결과가 어땠는지 사이클이 완성되는 거 같아."

사이클은 매수가 아니라 **선정**에서 시작한다. 각 단계의 "사유"를 기록해 한 종목·하루를 통째로 추적한다:

| 단계 | 사유 (기록) | 출처 |
|---|---|---|
| **① 선정** | 왜 오늘 이 종목이 후보가 됐나 — 등락률 3위·거래대금 상위·LLM 반도체강세·TSI+42 | S3/S4 |
| **② 모니터링** | 현재 어떤 조건/그룹이 충족 중인가 (실시간 준비도) | S6 |
| **③ 매수** | 어떤 그룹이 발화했나 + 그때 모든 조건 상태·시장맥락 | S6/태깅 |
| **④ 결과** | 손익·승패·보유시간·청산사유 | S8/S10 |
| **⑤ 개선(다음)** | **고르지 말아야 할 선정사유 · 끄야 할 매수조건**을 EV로 도출 → 설정 반영(weight↓/비활성) | S10/학습루프(기존) |

> **⑤는 기존 프로세스에 디테일을 더하는 것:** EOD 학습루프(`consolidate_and_apply`·`learning_memories`·설정 자동반영)는 이미 있다. 새로 만드는 게 아니라, 거기에 **조건/선정기준 단위의 EV 가지치기**(negative knowledge — "이 조건은 EV음수 → 끈다")를 얹는다. 학습의 진짜 힘은 *뭘 할까*보다 **뭘 피할까**에서 나온다.

→ **선정사유까지 태깅**하면 "어떤 *선정기준*으로 뽑힌 종목이 이기나"(등락률순위 vs 거래대금 vs LLM강세)도 EV로 비교 가능. "올바른 시장상황 파악"의 핵심.

> **재사용 — 선정 인프라는 이미 있다(Funnel):** S3/S4 결과·Funnel Monitor가 후보별 점수·순위·LLM 사유를 이미 생성/표시한다. 따라서 ①선정은 **새로 만들지 않고 기존 funnel을 재사용**한다. 이 엔진에서 *선정 측의 net-new* 는 단 하나: **등락률(상승률) 순위 API를 S3 소스로 추가**(거래량·거래대금 + 등락률) → 강세주 합류. 나머지(②~④)가 진짜 신규 작업.

## 핵심 개념 — 설정 가능한 조건 프레임워크

### 원자 조건 (atomic condition)
하나의 측정 가능한 매수 조건. 설정값(임계치)을 Settings에서 조정.
- 예: `체결강도 ≥ 55%`, `틱거래량배수 ≥ 2.0`, `등락률 1.5~5%`, `일봉 TSI > 0`, `당일고가 돌파`, `VWAP 상단`, `눌림 후 반등(10초 양봉)`, `10초봉 N연속 상승`, `시간창 09:30~15:00`.
- 각 조건: `{id, name, type, params(json), enabled}`.

### 조건 그룹 (condition group = 매수 전략)
원자 조건들을 **AND로 묶은** 하나의 명명된 전략.
- 예: `돌파전략 = (당일고가 돌파) AND (체결강도≥55%) AND (틱거래량≥2×) AND (VWAP 상단)`
- 예: `눌림전략 = (급등후 VWAP 조정) AND (10초 양봉 반등) AND (체결강도 회복)`
- 각 그룹: `{id, name, condition_ids[](AND), enabled, weight(가중), assigned_to(regime/profile)}`.

### 전체 매수 판정 (OR)
```
매수신호 = OR( 활성 그룹들 중, 현재 레짐/프로파일에 할당된 그룹이 발화 )
        = 그룹A(AND…) OR 그룹B(AND…) OR …
```
- **탐색모드:** 거의 모든 그룹 활성 + OR → 매수 폭증.
- **수렴 후:** EV 낮은 그룹 weight↓/비활성 → 정밀.

### 레짐/Risk Profile 할당
각 그룹을 레짐(Risk On/중립/방어/변동성) 또는 Risk Profile에 할당 → "어떤 시장상황에 어떤 전략" 매핑을 데이터로 학습·운영.

### 🔑 태깅 (가지치기를 가능케 하는 전부)
매수 체결 시 1행 기록, 청산 시 결과 채움:
```json
{
  "order_id": "...", "symbol": "...", "trade_date": "...",
  "selection_reason": {                              // ① 왜 후보로 선정됐나 (S3/S4)
    "sources": ["등락률순위#3", "거래대금상위"],       //   어떤 소스가 surfacing
    "scores": {"universe_score": 0.36, "llm_suitability": 0.72, "일봉TSI": 42.0},
    "llm_note": "반도체 섹터 강세 모멘텀"
  },
  "fired_groups": ["돌파전략"],                      // ③ 무엇이 쐈나(OR 중 발화 그룹)
  "condition_states": {                              // 진입 순간 모든 원자조건 값(발화 여부 무관)
    "체결강도": 0.62, "틱거래량배수": 2.3, "등락률": 2.3,
    "일봉TSI": 11.0, "VWAP위치": "above", "돌파": true, "눌림": false, ...
  },
  "market_context": {"regime": "neutral", "market_tone": "negative",
                      "time_bucket": "10:30", "vix": 18.2},  // 어떤 상황
  "outcome": {"realized_pnl": -1700, "win": false, "hold_sec": 1820,
               "exit_reason": "stop_loss"}              // 어떻게 끝났나(청산 시 채움)
}
```
→ 이 통짜 기록으로 오프라인에서 **임의 조건/그룹/맥락별 승률·EV**를 다 계산 → 가지치기·시장상황별 전략 도출.

## 자본·포지션 (탐색 표본 극대화)
- **풀 예수금** 사용. WS 한도(41)까지 채워 ~40종목 소액 균등 배분.
- 빠른 회전: 청산 후 슬롯 즉시 재매수 → 하루 표본 극대화.
- 사이징: 탐색모드에선 `max_positions`↑(예 40), 예산률↑(예 0.95). (기존 baseline×rate/슬롯 재사용, 파라미터만 공격적)

## 측정·가지치기 (Phase 3, 적응) — 기존 학습루프에 디테일 추가
- 지표: **기대값 EV = 승률 × 평균이익 − 패율 × 평균손실** (승률만이 아니라 손익비 포함).
- **선정기준·그룹·조건·맥락별** EV 집계 (표본 ≥ N, 예 30).
- **가지치기 우선(negative first):** EV 음수인 **선정사유/매수조건을 끄는 것**부터 — "고르지 말아야 할/사지 말아야 할" 도출이 수렴의 지름길.
- **자동 가중:** EV 높은 그룹 weight↑, 낮은 그룹 weight↓. weight 0(완전차단)은 표본 매우 많고 지속 부진할 때만(운에 좋은 전략 안 죽임).
- **기존 재사용:** 집계·반영은 기존 EOD 학습루프(`consolidate_and_apply`·`learning_memories`·설정 자동반영)에 **조건/선정 단위 EV 디테일만 얹는다**. 새 파이프라인 아님.

## 청산 (기존 유지)
레짐별 손절·트레일링·15:20 강제청산. 청산사유(exit_reason)는 태깅 outcome에 기록.

## UI (PM 요구 반영)

### Trade History
각 거래 행에 **선정사유**("등락률#3·거래대금상위·LLM:반도체강세") + **매수사유**(발화 그룹 + 핵심 신호 "돌파·체결강도62%·거래량2.3×") + **매도/손절사유**(stop_loss/trailing/take/EOD) + 손익. 클릭 시 태깅 전체(선정·조건상태·맥락) 상세 → **"왜 골랐고·왜 샀고·어떻게 끝났나" 한눈에.**

### Trading Monitor (실시간)
매수 대기 종목에 **선정 사유**(왜 오늘 후보인지) + **현재 충족 조건/그룹**(실시간 준비도 확장) 표시. 매수 발생 시 즉시 매수사유 노출. 보유 포지션에 손절선·트레일링 사유.

### Settings — 매수조건 설정
- 원자 조건별 **설정값 편집**(체결강도 임계치 등).
- **그룹 빌더:** 조건들을 AND로 묶어 그룹 생성/편집, enabled 토글.
- **할당:** 그룹을 레짐/Risk Profile에 배정.

### Review (EOD 분석)
- **선정기준별·그룹별·조건별·시장맥락별 EV/승률 표** + 가지치기 추천("돌파전략 EV음수 30표본 → weight↓", "등락률순위 선정 종목 EV > 거래대금 선정").
- → 매수전략뿐 아니라 **어떤 선정기준이 이기는지**도 도출.
- Missed/False와 통합(기존 학습루프 연계).

## 안전장치 (필수)
- **🔒 모의 전용 게이트:** 탐색모드(OR 폭증·풀예수금)는 **KIS virtual일 때만** 허용. 실계좌면 하드 차단(실수로 80패가 실손실 방지).
- **학습루프 과민반응 방지:** 탐색 중 고손실일(20승80패)은 의도된 것 → EOD가 "필터 너무 느슨"으로 자동 방어전환하지 않음. 탐색모드 플래그가 켜진 동안은 데이터 수집만, 가지치기는 표본 충분 시.
- 등락률 상한·수직급등 회피는 가드레일로 유지(잡주 추격 한도).

## 검증 (HARD 제약)
- 과거 틱/체결강도 데이터 미저장 → **10초봉 백테스트 불가**.
- 검증 = **모의 포워드 측정**: 켜고 며칠~몇 주 데이터 축적 → 그룹/조건별 EV가 드러남 → 가지치기. (보조: 1분봉 근사 백테스트로 로직 sanity)
- 이 방식은 모의계좌라 가능. "내일 20승80패 → 두 달 뒤 80승20패"는 이 사이클의 목표 곡선.

## 빌드 단계 (점진 — 각 단계가 독립 실행 가능)

- **Phase 1 — 조건 프레임워크 + 태깅 (데이터 기반):**
  원자 조건/그룹 데이터모델, OR 매수 판정, WS 10초봉·체결강도·VWAP 인프라, 통짜 태깅, 모의전용 게이트, 풀예수금·다종목 사이징.
- **Phase 2 — UI:** Trade History/Monitoring 사유 표기, Settings 조건·그룹·할당 편집.
- **Phase 3 — 리뷰 분석 + 자동가중:** 그룹/조건/맥락별 EV 집계·표시, 가지치기 추천, 자동 weight 조정.

> Phase 1이 데이터를 만들고, Phase 2가 운영/편집, Phase 3가 수렴 엔진. Phase 1 없이는 나머지가 무의미하므로 Phase 1부터.

## 확정 결정 (PM, 2026-06-06)
1. 진입 조건 관계 = **모두 OR** (탐색 폭증, 추후 가지치기).
2. 풀 예수금 + 하루 다종목.
3. 매수/손절 사유 UI 표기 + 리뷰 분석.
4. Settings에서 조건 설정값·AND 그룹핑·레짐/프로파일 할당.
5. 모의계좌 전용(실계좌 차단).
6. 측정 지표 = EV(승률×손익비), 자동 가중.
