# 매수 적극성 향상 — 종합 개선 설계서 v1.0 (확정)

작성일: 2026-06-03 · 대상: S4 스크리닝 / S6 매수 게이트

## 원본 요구사항 (PM)

> 매수가 적극적으로 이루어지지 않는다. 유니버스·S5가 너무 타이트한가? 매수조건감시는
> 거래량 추이·TSI를 쓰나? → 분석 결과: (1) S4 [:30] 캡 + LLM 보수 선별로 후보가 ~6개,
> (2) 게이트가 등락률 3~5% 좁은 밴드 + 거래량 스냅샷, TSI 미사용. PM 지시:
> ① 후보 캡을 WS 한도(~41) 가깝게 채우고 ② 등락률 하한 1.5% ③ TSI를 게이트+랭킹 둘 다.

## 핵심 제약 — KIS WebSocket 모니터링 한도

- KIS 실시간 시세 WebSocket(H0STCNT0)은 **1세션 최대 41종목** 등록 가능(KIS 정책, 웹검색 확인).
  - 단, **docs/reference 매뉴얼(xlsx/md)엔 이 수치가 명시돼 있지 않음** → 설계에선
    **설정 상수 `WS_MAX`(기본 41)** 로 두고 운영 중 조정. 정확값은 **장중 실측**으로 확정 권장.
- 보유 포지션도 같은 WS로 감시되므로, **후보 모니터링 가용분 = WS_MAX − 보유 포지션 수**.
- 현재는 S4 후보(~6)만 구독 → **WS 용량의 1/7만 사용**. 채울 여지가 큼.

## 현재 병목 (분석 결과)

```
유니버스 60 ──[:30] 하드캡──▶ 30 ──LLM 보수선별──▶ ~6 ──WS 구독 6 ──S6 게이트──▶ ~1 매수
            (절반 버림)              (입력 늘려도 ~6)   (41 중 6만)   (3~5% 좁은밴드, TSI없음)
```

## 개선 설계 (3축)

### ① WS 한도까지 후보 채우기 (S4)

문제: LLM은 입력을 늘려도 "엄선한 소수"만 반환 → 캡만 올려선 안 채워짐.

설계: **LLM 정성 선별 + 정량 top-up 하이브리드.**
1. `[:30]` 입력 캡 → **전체 유니버스**로 확대 — LLM이 더 넓게 검토.
2. LLM이 고른 후보(고확신) + **블렌드 점수 상위 종목으로 정량 top-up** →
   **모니터링 목표 = `min(40, WS_MAX − 보유포지션수)`** 까지 채운다. (고정 상한 40)
   - LLM 후보: 우선 배정(suitability 높음).
   - 부족분: 블렌드 점수 상위로 채우되 **일봉 TSI > 0(상승추세) 종목만** — 하락추세
     종목을 자리만 채우려고 넣지 않는다. 상승추세 종목이 부족하면 40 미만이어도 OK.
   - **블렌드 점수 = 0.4×TSI_norm + 0.4×유니버스점수_norm + 0.2×LLM_suitability**
3. WS 구독은 이 확장된 후보 전체. (S6 게이트가 그 안에서 실제 매수 선별)

> 모니터링은 넓게(≤40), 매수는 좁게(레짐별 max_positions 2~12). "넓게 보고, 추세 종목만, 선별 매수".

### ② 등락률 하한 완화 (S6 게이트)

- `engine.min_price_change_pct` **3.0% → 1.5%** (진입 밴드 1.5~5%).
- 효과: 이미 3% 급등한 종목 추격 대신, **초기 모멘텀(1.5~3%) 구간 진입**.
- 약신호 유입 우려는 ③ TSI 게이트로 상쇄.

### ③ TSI (True Strength Index) — 게이트 + 랭킹 (둘 다)

**TSI 계산** (모멘텀의 추세 강도·방향):
```
mtm = close − prev_close
TSI = 100 × EMA(EMA(mtm, r), s) / EMA(EMA(|mtm|, r), s)
v1 기간: 표준 r=25, s=13 (일봉)
입력 바: 일봉 종가 (장전 계산). 최소 ~30거래일 종가 필요.
```
- **TSI > 0** = 상승 추세, **TSI < 0** = 하락 추세, 절댓값 = 추세 강도.

**(a) 게이트 — 필수 조건:** 기존 `volume_ratio + price_change + time_window` 에
**`일봉 TSI > 0` (추세 살아있음 확인)** AND 추가. 등락률 하한을 1.5%로 낮춘 대신,
**추세가 꺾인 종목(TSI<0)은 매수 안 함** → 떨어지는 칼·데드캣 바운스 회피.

**(b) 랭킹 — 점수 가중:** 후보 선정·우선순위의 블렌드 점수에 TSI 0.4 가중(위 ① 참조).
높은 TSI = 강한 상승추세 우선 구독·매수.

## 구현 노트 (TSI 데이터 소스 — v1 확정)

- **v1 = 일봉 TSI(25/13), 장전 1회 계산.** 랭킹·게이트 모두 이 값 사용.
  - 이유(트레이더 관점): 단타 게이트의 "지금 오르는가"는 이미 등락률·거래량이 포착.
    TSI는 "추세가 살아있는 종목만 진입"하는 품질 필터 역할이 정석. 일봉 TSI면 장전에
    한 번 계산해 캐시 → 실시간 부담 0.
  - 데이터: KIS 일봉 차트(get_daily_chart) 종가 ~30일. 기존 technical_indicators 확장.
  - TSI 데이터 없으면(신규상장 등) **차단 아니라 통과**(정량 게이트로만) — 결손이 매수를 막지 않음.
- **v2(후속):** WS 틱 스트림을 분봉으로 집계해 **인트라데이 TSI** 추가 → 장중 추세 가속 포착.

## 안전장치

- WS 구독은 **41 하드 가드**(초과 구독 차단). 보유 포지션 우선, 잔여를 후보에 할당.
- 등락률 1.5%는 `engine.min_price_change_pct` floor로 clamp 유지(가드레일).
- TSI 게이트는 데이터 없으면(분봉 미수집) **차단이 아니라 통과**(기존 정량 게이트로만) — 데이터 결손이 매수를 막지 않게.

## 확정 결정 (PM 위임 → 전문 트레이더 관점, 2026-06-03)

1. **모니터링 목표 수** — **고정 상한 40** (`min(40, WS_MAX − 보유수)`). WS 41 중 1 헤드룸.
2. **TSI 게이트 데이터** — **v1 = 일봉 TSI(25/13), 장전 계산.** 게이트(>0)+랭킹 둘 다.
   실시간 분봉 TSI는 v2. (근거: 단타 진입 타이밍은 등락률·거래량이, 추세 품질은 TSI가)
3. **top-up 점수식** — **0.4×TSI + 0.4×유니버스점수 + 0.2×LLM확신.** 모멘텀·유동성 동등, LLM 타이브레이커.
4. **등락률 상한** — **5% 유지(상향 안 함).** 적극성은 하한(1.5%)으로만 확대. 과매수 추격 금지.

### 트레이더 관점 종합 논리
> "그물은 넓게(40종목 감시), 진입은 일찍(1.5%부터), 단 **추세가 살아있는 종목만**(TSI>0),
> 과열은 피한다(≤5%)." → 더 많은 기회를 보되 falling knife와 고점 추격을 동시에 차단.
> 거래량 2.5배(기존)는 모멘텀 확인용으로 유지.
