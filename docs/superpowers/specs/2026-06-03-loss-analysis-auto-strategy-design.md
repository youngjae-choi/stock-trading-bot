# 손실 분석 → 전략 자동반영 — 설계서 v1.1

작성일: 2026-06-03 · 대상 화면: False Positive(손실분석)

## v1.1 변경 (PM 지시 2026-06-03)

손실분석 결과를 **즉시 직접 반영하지 않는다.** 흐름을 다음으로 변경:

```
장마감(S9) → Missed 분석 → False 분석 → Review 리포트에 종합
            → Review 종합 단계에서 Missed+False+당일거래를 합쳐 하루 한 번 일괄 자동반영
```

- "손실분석 실행" 버튼 = **분석·제안 미리보기 전용**(반영 안 함). 원인 서술 + "EOD에 반영될 전략" 제안을 팝업에 표시.
- 실제 설정 반영 = **EOD Review 종합(S10/S11)** 이 Missed 제안과 합쳐 충돌 조정 후 1회. reviewed 숨김도 이때.

## 원본 요구사항 (PM 발화 그대로 인용)

> 손실분석결과가 저렇게 나오는게 맞아?? 내가 원하는건 손실난 종목을 모아서 표현한다.
> 손실분석버튼을 누르면 각 종목별 손실원인을 분석하고 재발방지하기위한 추가 매매전략이
> 있는지 표시한다. 손실분석을 위한 표본이 더 필요한경우 분석을 거부한다. 분석이 완료되면
> 전략은 자동으로 시스템에 반영 후 사용자에게 알리고 분석이 끝난 데이터는 미 표시한다.

## 왜 필요한가 (WHY)

현재 "손실분석 실행" 버튼은 손실 거래를 `false_positive_cases`에 **기록만** 한다
(`generate_false_positives_for_date`). 원인 심층분석·전략 생성·시스템 반영이 없다.

추가로 현재 동작에는 **버그**가 있다: generate는 단일 날짜(오늘)만 분석하는데 화면 목록은
날짜 범위를 조회한다. 그래서 팝업은 "전체 손실 0건"인데 목록엔 1건이 보이는 불일치가 난다.

다행히 이미 **학습 메모리 시스템**(`run_learning_memory_builder`)이 손실/미진입에서
전략(action)을 생성하고 표본 기반 자동적용 여부를 판단하는 엔진을 갖고 있다. 단 EOD에만
돈다. 본 기능은 이 버튼을 그 엔진(을 개량한 버전)에 연결해 "손실 → 원인 → 전략 →
자동학습"의 닫힌 루프를 사용자 주도로 돌리는 것이다.

## 핵심 동작 (확정 결정 반영)

| 단계 | 동작 |
|---|---|
| 1. 표시 | 손실난 종목 목록 — **분석 안 된(미reviewed) 것만** 표시 |
| 2. 실행 | "손실분석 실행" → **화면 범위 내 미분석 손실 전체**를 대상으로 분석 |
| 3. 표본 게이트(전역) | **전체 손실 < 3건 → 분석 거부**. 아무 것도 바꾸지 않고 거부 사유 표시 |
| 4. 원인분석(하이브리드) | 통과 시 **각 종목별 손실 원인을 LLM이 서술** + 후보 전략 제안 |
| 5. 전략 제안(결정적) | 같은 원인 패턴 3건+ 전략은 "EOD 반영 대상", 미만은 "관찰 중". **버튼은 제안만, 반영 안 함** |
| 6. 버튼 팝업 | **완료 팝업**에 제안 전략(EOD 반영 예정) + 관찰 보류 + 거부 사유. (텔레그램/Alert 미사용) |
| 7. EOD 종합 반영 | 장마감 Review 종합 단계가 **Missed+False+당일거래**를 합쳐 충돌 조정 후 **하루 1회 일괄** 설정 반영(가드레일 clamp+감사). 승인 단계 없음 |
| 8. 미표시 | **EOD 반영 시점에** 분석된 손실 case → reviewed 처리 → 목록에서 숨김 |

## 결정 사항 (PM 승인 완료)

- **분석 방식:** 하이브리드 — LLM 원인 서술 + 결정적 action 반영.
- **자동반영 범위:** 표본 통과 전략은 소/대손실 구분 없이 **전부 자동반영**.
- **표본 게이트:** 전역(총 손실 3건 미만 거부) + 패턴별(원인 패턴 3건+ 만 반영) **둘 다**.
- **AI confidence 액션:** `raise_ai_confidence_min` **제외**(2026-06-02 AI confidence 게이트 폐지로 무효).
- **전략 대상 설정:** 시스템의 **모든 튜닝 가능한 설정값을 자유롭게 조정** 가능.
- **알림:** **완료 팝업만**.

## 아키텍처

**A. 미리보기 — `POST /api/v1/false-positive/analyze?start=&end=` (반영 안 함)**

```
[손실분석 실행 버튼]
  └─ POST /false-positive/analyze (preview=true)
       1) 대상 수집: false_positive_cases 중 (범위, reviewed_at IS NULL) 손실
       2) 전역 표본 게이트: 총 손실 < 3 → {refused:true, needed:3}
       3) (옵션) LLM 원인 서술
       4) 전략 제안 도출(결정적 게이트): 패턴 그룹핑 → 패턴 ≥ 3 & 화이트리스트면
          "proposed", 아니면 "observing". 값은 가드레일 clamp.
       5) return {proposed:[...], observing:[...], refused:false}  ← 반영/숨김 안 함
  └─ 완료 팝업: EOD 반영 예정 전략 N건 / 관찰 M건 / (거부 시) 사유
```

**B. EOD 종합 반영 — Review 파이프라인 (job_review_audit, 자동)**

```
15:20 S9 청산
 16:00 job_review_audit:
    - generate_false_positives(today)  (기존)
    - run_review_audit(today): 리포트에 False+Missed+당일거래 종합 (기존)
 → 종합 반영 단계 (신규/S11 확장):
    1) Missed 제안 + False 제안 도출 (loss_strategy.derive_strategies 재사용)
    2) 충돌 조정: 같은 설정 키는 더 보수적(손실 방어적) 값 채택
    3) 일괄 자동반영: clamp → settings_store.upsert_setting + audit + 학습메모리
    4) 반영에 쓰인 False case → reviewed 처리(숨김)
```

핵심: **반영·숨김은 B(EOD)에서만.** A(버튼)는 같은 도출 로직으로 "제안 미리보기"만 한다.

### 모듈 경계 (단일 책임)

- **`loss_analysis.py` (신규 서비스):** 오케스트레이션. 수집 → 게이트 → LLM → 전략 → 반영 → reviewed.
- **`false_positive.py` (기존):** case 수집/조회/reviewed 처리 재사용.
- **`learning_memory.py` (기존, 개량):** 패턴 그룹핑 + 표본 카운팅 로직 재사용. 고정 action
  카탈로그는 "자유 설정 타깃"으로 대체하고 `raise_ai_confidence_min`은 제거.
- **튜닝 허용 설정 화이트리스트:** "자유롭게 조정"하되 자동반영 가능한 설정 키 목록을 코드에
  명시(손절/트레일링/등락률범위/거래량배수/프로파일비중·보유수/진입시간창 등 매매 파라미터).
  운영 비매매 설정(스케줄·토큰 등)은 제외해 사고 방지.
- **`settings_store.upsert_setting` (기존):** 자동반영 = 설정 변경 + audit_events 기록(되돌리기 가능).
- **LLM 호출:** 기존 daily_plan의 anthropic provider 재사용.

## 안전장치 (모든 설정 자유 + 자동반영의 리스크 완화)

1. **가드레일 clamp:** 적용값은 기존 Settings 가드(예: `engine.min/max_price_change_pct` floor/ceil,
   `position_size_pct ≤ 30%`)로 강제 제한. LLM이 극단값을 제안해도 안전 범위로 잘림.
2. **패턴 표본 ≥ 3:** 우연한 1~2건 손실로 시스템이 흔들리지 않음.
3. **감사·되돌리기:** 모든 자동반영은 `audit_events`에 old→new 기록(actor=loss_analysis).
4. **거부 우선:** 표본 부족이면 아무 것도 바꾸지 않음.

## 상태별 UI & 메시지 (완료 팝업)

- **거부:** "분석 거부 — 손실 표본 부족 (현재 N건 / 최소 3건 필요). 더 쌓인 뒤 다시 시도하세요."
- **성공:** "분석 완료 — 반영된 전략 N건 / 관찰 보류 M건. (반영 목록 나열)"
- **반영 0건(통과했으나 패턴 미달):** "분석 완료 — 즉시 반영할 전략 없음. 관찰 보류 M건."

## 엣지케이스

- 손실 0건: 버튼 비활성 또는 "분석할 손실 없음".
- 이미 전부 reviewed: 목록 빈 상태 + 버튼 "분석할 신규 손실 없음".
- LLM 실패: 서술은 생략하되 **결정적 전략 도출/반영은 계속**(LLM은 보조 서술 용도).
- 동일 설정에 상충하는 두 전략: 더 보수적인(손실 방어적인) 값 채택.
