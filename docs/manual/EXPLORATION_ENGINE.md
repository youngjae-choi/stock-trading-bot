# 탐색 엔진 운영 매뉴얼 (Exploration Engine)

> 최종 수정: 2026-06-06
> 대상: 운영자(PM) + Sisyphus(Orchestrator)
> 설계 원본: [`docs/superpowers/specs/2026-06-06-exploration-buy-strategy-engine-design.md`](../superpowers/specs/2026-06-06-exploration-buy-strategy-engine-design.md)
> 본 문서는 **탐색 엔진을 어떻게 켜고·끄고·모니터링하고·읽는가**를 정의한다. 코드 내부 구현이 아니라 운영 관점이다.
> 시스템 전체 흐름은 [`SYSTEM_GUIDE.md`](SYSTEM_GUIDE.md), 일일 시간표는 [`OPERATION_SPEC.md`](OPERATION_SPEC.md) 참조.

## 1. 개요 & 철학

탐색 엔진은 **모의계좌 샌드박스에서 "그물을 최대로 던지는" 매수 모드**다.
모든 매수 조건을 OR로 묶어 하루에 가급적 많은 종목을 매매하고, 매수마다 "왜 골랐고·왜 샀고·어떻게 끝났나"를 통째로 기록한다. 쌓인 데이터로 기대값(EV) 낮은 조건/그룹을 가지치기해 점점 정밀한 전략으로 수렴시킨다.

```
탐색(explore) → 측정(measure) → 가지치기(prune) → 수렴(converge)
```

목표 곡선: **"내일은 20승 80패라도, 한두 달 뒤에는 80승 20패"**.

### ⚠️ 초기 고손실은 의도된 것 — 버그가 아니다

탐색모드를 켜면 처음 며칠은 승률이 낮고 손실이 누적되는 것이 **정상**이다. 이것은 데이터를 모으기 위한 의도된 단계다. 시스템은 이 기간을 "필터가 너무 느슨하다"고 오판해 자동으로 방어 전환하지 않는다(아래 §11 안전장치 참조). 실계좌에서는 불가능한 실험을 **모의계좌에서만** 수행한다.

## 2. 한 종목·하루를 통째로 추적하는 5단계 사이클

사이클은 매수가 아니라 **선정**에서 시작한다. 각 단계마다 "사유"를 기록한다.

| 단계 | 무엇을 기록하나 | 담당 |
|---|---|---|
| **① 선정** | 왜 오늘 이 종목이 후보가 됐나 (등락률순위·거래량순위·거래량급증·LLM 메모·일봉TSI) | S3 모멘텀 / S4 |
| **② 모니터링** | 현재 어떤 조건/그룹이 충족 중인가 (실시간 준비도) | S6 BarEngine |
| **③ 매수** | 어떤 OR 그룹이 발화했나 + 그 순간 모든 조건 상태 + 시장맥락 | S6 / 태깅 |
| **④ 결과** | 손익·승패·보유시간·청산사유 | 청산(기존) |
| **⑤ 개선** | EV로 "끄야 할 조건·고르지 말아야 할 선정기준" 도출 → 설정 반영 | EOD EV 가지치기 |

⑤는 **새 파이프라인이 아니다.** 기존 EOD 학습 루프(S10 Review)에 "조건/선정 단위 EV 가지치기" 디테일을 얹은 것이다.

## 3. ① 종목 선정 (S3 단타 모멘텀)

S3 유니버스 선정이 **단타 모멘텀** 모델로 바뀌었다.

### 소스 (2종 — 거래대금 제거)
- **거래량 순위** (KIS `get_volume_rank`)
- **등락률(상승률) 순위** (KIS `get_price_rank`)

> 과거의 **거래대금 순위 소스는 제거**됐다. 강세주(상승률 + 거래량 급증) 합류가 목적.

### 점수 (가중 합산 50:50)
- **상승폭(등락률) 점수 50%** — 등락률 순위가 있으면 순위 점수, 없으면 등락률 정규화.
- **거래량급증 점수 50%** — `vol_inrt`(전일대비 거래량증가율 %)를 `/300`으로 정규화(전일 대비 3배 = 1.0 포화). 결손 시 거래량 순위 점수로 폴백.

> 점수 가중치는 시장 톤(positive/neutral/negative/mixed)별 테이블로 분리돼 있으나 **현재는 전 톤 모두 50:50** 동일하다.

### 유지되는 제외 규칙 (sanity cut)
- 거래량 0 / 가격 0원 종목 제외
- 변동률 ±29% 초과(상·하한가 추정) 제외
- **ETP 제외**: ETF/ETN/인버스/레버리지/2X·3X/단일종목 (이름·코드 패턴)
- **우선주 제외**: 종목명이 `우`/`N우`/`우B`/`우C` 등으로 끝남
- **스팩 제외**

KIS에서 최대 60종목 수집 → 점수순 상위 30종목을 DB 저장(`universe_filter_results`).

## 4. ③ 매수 — OR 조건 그룹

탐색모드가 켜지면 매수 판정이 다음으로 바뀐다:

```
매수신호 = OR(활성 그룹들 중 하나라도 발화)
        = 그룹A(조건들 AND) OR 그룹B(조건들 AND) OR …
```

각 그룹은 원자 조건들을 **AND**로 묶은 하나의 명명된 전략이다. 그룹이 많고 OR이므로 탐색모드에서는 매수가 폭증한다.

### 시드된 기본 그룹 4종

| 그룹명 | 구성 조건 (AND) |
|---|---|
| **돌파전략** (`grp_breakout`) | 당일고가 돌파 · 체결강도 55%+ · 틱거래량 2배+ · VWAP 상단 |
| **눌림전략** (`grp_pullback`) | 눌림 후 반등 · 체결강도 55%+ · VWAP 상단 |
| **모멘텀전략** (`grp_momentum`) | 10초봉 3연속 상승 · 체결강도 55%+ · 틱거래량 2배+ |
| **베이스라인(기존게이트)** (`grp_baseline`) | 등락률 1.5~5% · 일봉 TSI>0 · 시간창 09:30~15:00 |

> "베이스라인"은 기존 게이트를 그룹으로 표현한 것 — 탐색모드에서도 기존 로직이 OR 한 갈래로 살아있다.

### 원자 조건 (임계치는 Settings에서 편집)

| 조건 | 의미 | 기본 임계치 |
|---|---|---|
| 당일고가 돌파 (`day_high_breakout`) | 현재가가 전일 일봉 고가 돌파 | `buffer_pct: 0` (=돌파 즉시) |
| 눌림 후 반등 (`pullback_rebound`) | 급등(VWAP+3%) 후 VWAP 근접 되눌림 + 10초 양봉 반등 | — |
| 10초봉 N연속 상승 (`momentum_rising_bars`) | 마지막 10초봉이 N연속 상승 | `min_bars: 3` |
| 체결강도 (`chegyeol_gangdo_min`) | 매수 체결강도(shnu_rate 0~1) ≥ 임계 | `min: 0.55` (55%) |
| 틱거래량 배수 (`tick_volume_mult_min`) | 마지막 틱거래량 / 평균 틱거래량 ≥ 배수 | `min: 2.0` |
| VWAP 상단 (`vwap_above`) | 현재가가 당일 VWAP 위 | `margin_pct: 0` (=above) |
| 등락률 밴드 (`change_rate_band`) | 등락률이 밴드 안 | `min: 1.5, max: 5.0` |
| 일봉 TSI>0 (`tsi_positive`) | 일봉 TSI 양수 (결손이면 통과 — 차단 안 함) | `min: 0` |
| 시간창 (`time_window`) | 진입 허용 시간대 | `09:30~15:00` |

> `buffer_pct`/`margin_pct`/`tsi min`은 **no-op 기본값(0)** 이다 — 기본 상태에서는 기존 동작과 100% 동일하고, 운영자가 0보다 큰 값을 주면 그만큼 더 엄격해진다(튜닝 노브).

### 풀 예수금 사이징
탐색 허용 시(아래 §9 게이트 통과 시) 사이징이 공격적으로 바뀐다:
- `exploration.budget_rate` = **0.95** (예수금 95% 사용)
- `exploration.max_positions` = **40** (최대 40종목 소액 균등 배분)

탐색이 아니면 기존 baseline 사이징(`daily_capital` 예산률 + 기존 `max_positions`)을 쓴다.

### 라이브 신호 산출 (10초봉 엔진)
WS 실시간 틱(H0STCNT0)을 종목별 **10초 OHLCV 봉**으로 집계하면서 다음을 라이브로 계산해 조건 `state`를 채운다:
- 체결강도(shnu_rate), VWAP, 당일고가 돌파, 눌림 반등, 10초봉 연속 상승, 틱거래량 배수, 등락률.
- 일봉 TSI는 외부 주입(라이브 엔진이 만들지 않음).

> 과거 틱/체결강도는 저장되지 않아 **10초봉 백테스트는 불가능**하다. 검증은 모의 포워드 측정으로만 한다(켜고 며칠~몇 주 데이터 축적 → EV가 드러남).

## 5. ④ 태깅 — 가지치기를 가능케 하는 전부

매수 체결 시 `trade_entry_tags`에 1행 기록하고, 청산 시 `outcome`을 채운다.

| 필드 | 내용 |
|---|---|
| `selection_reason` | ① 선정사유 — `sources`(예: 등락률순위#3·거래량순위#12·거래량급증) + `scores`(universe_score·llm_suitability·change_rate·일봉TSI) + `llm_note` |
| `fired_groups` | ③ OR 중 실제 발화한 그룹명 리스트 |
| `condition_states` | 진입 순간 **모든** 원자조건 값(발화 여부 무관) |
| `market_context` | `regime`·`market_tone`·`time_bucket`·`vix` |
| `outcome` | ④ 청산 시 채움 — `realized_pnl`·`win`·`hold_sec`·`exit_reason` |

이 통짜 기록 덕분에 EOD에 **임의의 조건/그룹/선정소스/맥락별 승률·EV**를 오프라인으로 다 계산할 수 있다.

## 6. ⑤ EV 가지치기 (측정 → 수렴)

EOD(S10 Review)에 자동 실행된다. 멀티데이 태그(기본 lookback 10일)를 모아 차원별로 EV를 집계한다.

### 지표
```
EV = 승률 × 평균이익 − 패율 × 평균손실
```
승률만이 아니라 **손익비를 포함**한다(작게 자주 이기고 크게 한 번 지면 EV는 음수).

### 집계 차원
- `fired_group` (발화 그룹별)
- `selection_source` (선정소스별 — 등락률순위 vs 거래량순위 vs 거래량급증)
- `regime` (레짐별)

### 가지치기 규칙 (negative-first)
- 표본 **n ≥ 30** AND **EV < 0** 인 대상만 추천한다(표본 부족하면 손대지 않음).
- 기본 액션 = **downweight**: 그룹 `weight` × 0.5 (단, **floor 0.1** — 완전 0으로 죽이지 않음. 운 좋은 전략 보호).
- 표본이 매우 크고(n ≥ 90) 지속 음수일 때만 **disable**: weight를 floor로 내리고 `enabled=0`.
- 그룹이 아닌 차원(선정소스/레짐)은 weight 대상이 아니라 메모리 기록(negative-knowledge)만 한다.

### 결과
- `condition_groups.weight` 자동 조정(`apply=True`로 실행됨).
- `learning_memories`에 "사지/고르지 말아야 할" 요약 1행(category=`ev_pruning`) — 다음날 선정/매수 컨텍스트 참고.

> 핵심 철학: 학습의 진짜 힘은 *뭘 할까*보다 **뭘 피할까**에서 나온다.

## 7. ② 청산 (기존 그대로)

탐색 엔진은 청산 로직을 바꾸지 않는다. 기존 그대로:
- 레짐별 손절 / 트레일링 스탑
- 15:20 EOD 강제 청산(S9)

청산사유(`exit_reason`: stop_loss / trailing / take / EOD)는 태깅 `outcome`에 기록돼 EV 분석에 쓰인다.

## 8. 운영: 켜기 / 끄기

### 켜기/끄기 (Settings 탐색모드 카드)
- 콘솔 **Settings → 탐색모드 카드**에서 ON/OFF.
- 내부 설정 키: `engine.exploration_mode` (기본 **OFF**).
- 같은 카드에서 `exploration.budget_rate`(기본 0.95), `exploration.max_positions`(기본 40) 편집.

### 🔒 모의계좌 하드 게이트
설정을 ON으로 해도 **KIS 모의투자(openapivts)일 때만** 실제로 활성화된다.
실계좌(`_is_virtual_trading()=False`)면 설정 무관하게 **무조건 차단**된다(`exploration_gate.is_exploration_allowed()`). 실수로 80패가 실손실이 되는 것을 막는 안전장치다.

### 조건/그룹/임계치 편집
- 원자 조건 임계치(체결강도·틱거래량·등락률 밴드 등) 편집.
- 그룹 빌더: 조건을 AND로 묶어 그룹 생성/편집, `enabled` 토글.
- 레짐/프로파일 할당(`assigned_to`)도 편집 가능.

> **레짐 할당은 현재 미적용**이다. `assigned_to`를 설정해도 매수 평가는 현재 **활성 그룹 전부**를 OR로 본다(레짐 필터링 없음). 즉 지금은 켜진 그룹이 모두 적용된다.

## 10. 안전장치

- **🔒 모의 전용 하드게이트** — 실계좌면 설정 무관하게 차단(위 §8).
- **학습 루프 과민반응 방지** — 탐색 중 고손실일(20승80패)은 의도된 것이므로, 표본이 충분(n≥30)해질 때까지 가지치기하지 않고 **데이터 수집만** 한다. EOD가 "필터가 너무 느슨"하다고 자동 방어 전환하지 않는다.
- **weight floor 0.1** — 음수 EV 그룹도 완전히 죽이지 않는다(운 좋은 전략 보호). disable은 대표본(n≥90) 지속 음수일 때만.
- **TSI 결손은 통과** — 일봉 TSI 데이터가 없으면 차단이 아니라 통과(데이터 결손으로 매수를 막지 않음).

## 11. ⚠️ 주의사항

1. **초기 고손실은 정상이다.** 처음 며칠 손실 누적을 보고 끄지 말 것 — 데이터가 쌓여야 EV가 드러난다.
2. **실계좌 전환 전 반드시 OFF.** 탐색모드(OR 폭증·풀예수금)는 모의 전용이다. 실계좌로 넘어가기 전:
   - `engine.exploration_mode` = OFF 확인
   - EV로 검증돼 살아남은 그룹만 활성으로 좁히기
3. **레짐 할당은 아직 매수에 반영되지 않는다** — 그룹을 특정 레짐에만 적용하려는 의도라면 현재는 동작하지 않음(전부 적용). 의도치 않은 그룹은 `enabled=0`으로 끄는 것이 확실하다.
4. **검증 = 모의 포워드 측정뿐.** 10초봉/체결강도 과거 데이터가 없어 백테스트가 불가하다. 며칠~몇 주 켜서 측정해야 한다.
