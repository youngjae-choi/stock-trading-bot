# GLOSSARY.md — 용어 사전

AI Agent 준수 사항: 코드·주석·문서·로그에 이 용어를 일관되게 사용한다.
한국어/영어 혼용 시 아래 기준을 따른다.

---

## 역할 (Roles)

| 용어 | 영문 코드 | 설명 |
|------|-----------|------|
| PM | — | 비개발자 프로덕트 매니저. 기획·방향 결정 담당 |
| Sisyphus | — | Claude(AI)의 역할명. 메인 지휘자·게이트 관리자 |
| Oracle | — | Codex CLI의 역할명. 코드 리뷰·통합·테스트 |
| Frontend | — | Gemini CLI의 역할명. UI 전담 구현 |

---

## 도메인 용어

<!-- 프로젝트별로 도메인 용어를 기술한다. 아래는 채우기 예시. -->

| 용어 | 영문 코드 | 설명 |
|------|-----------|------|
| 항목 | `item` | 시스템이 관리하는 기본 데이터 단위 |
| 사용자 | `user` | 시스템에 로그인할 수 있는 계정 |
| 관리자 | `admin` | 사용자 관리, 삭제 권한을 가진 역할 |
| 일반 회원 | `member` | 기본 CRUD 권한만 가진 역할 |

## 상태 용어

<!-- 프로젝트의 상태값을 일관되게 정의한다. 아래는 채우기 예시. -->

| 상태 | 영문 코드 | 설명 |
|------|-----------|------|
| 활성 | `active` | 정상 사용 가능 상태 |
| 보관 | `archived` | 더 이상 사용하지 않지만 삭제하지 않은 상태 |
| 대기 | `pending` | 처리 중 또는 승인 대기 |

## 금지 용어 / 혼용 금지

<!-- 프로젝트 내 혼동되기 쉬운 용어를 정리한다. 아래는 채우기 예시. -->

| 금지 | 올바른 용어 | 이유 |
|------|------------|------|
| `data` (단독 변수명) | 구체적 이름 (`items`, `userList` 등) | 의미가 불명확 |
| `handle` (단독 함수명) | `handleSubmit`, `handleDelete` 등 | 행위가 드러나야 함 |
| `temp`, `tmp` (영속적 사용) | 목적에 맞는 이름 | 임시 변수가 영구 존재하면 안 됨 |
